## 0. v0.3 핵심 결정

이번 버전에서 캐릭터 개그의 기준을 아래처럼 확정한다.

> 이 게임의 코미디는 “마왕 혼자 웃기는 것”이 아니라, 서로 다른 직업 세계관을 가진 인물들이 같은 상황을 전혀 다르게 해석하면서 발생한다.

핵심 축은 세 개다.

| 축 | 관계 | 코미디 원리 | 캠페인 기능 |
|---|---|---|---|
| 핵심 운영 콤비 | 마왕 ↔ 바티 | 마왕은 거창하게 선언하고, 바티는 차갑게 현실로 번역한다. | 튜토리얼, HUD 경고, 전투 준비, 결산 |
| 핵심 라이벌 콤비 | 마왕 ↔ 레온 | 마왕은 악의 운영자 흉내를 내고, 레온은 용사 매뉴얼 흉내를 낸다. 둘 다 초짜라서 서로의 허세를 알아본다. | DAY 03, DAY 25, DAY 30 보스전 |
| 경제/보물 콤비 | 마왕 ↔ 골딘 ↔ 니아 | 마왕은 위엄을 말하고, 골딘은 손실을 계산하며, 니아는 보안 평점을 매긴다. | 보물 보관실, 도적 침입, 결산 |

이 중 `마왕-바티`, `마왕-레온`은 반드시 반복적으로 사용한다.
특히 바티는 단순 튜토리얼 캐릭터가 아니라, 마왕의 코미디를 완성하는 핵심 참모로 취급한다.

## 1. 캐릭터 코미디 설계 원칙

### 1.1 기본 대사 구조

대부분의 코믹 대사는 아래 3박자로 작성한다.

| 단계 | 기능 | 예시 구조 |
|---:|---|---|
| 1 | 허세/선언 | 마왕 또는 용사가 장르적으로 거창한 말을 한다. |
| 2 | 현실/규정/비용 | 참모, 회계, 성기사, 도적이 그 말을 실무적으로 해석한다. |
| 3 | 재허세/당황/역공 | 원래 말한 인물이 체면을 살리려고 더 웃긴 말을 한다. |

예시:

```text
PLAYER: 오늘부터 이 동굴은 공포의 성채다!
BATI: 성채 등록 기준에는 방이 최소 여섯 개 필요합니다. 현재는 축축한 원룸입니다.
PLAYER: ...공포의 원룸부터 시작한다!
```

### 1.2 코미디의 중심은 “관계”다

캐릭터별 개인 개그도 사용하지만, 반복 재미는 관계에서 나온다.

| 관계 | 충돌하는 가치 | 반복 개그 |
|---|---|---|
| 마왕 ↔ 바티 | 위엄 vs 실무 | 마왕의 선언을 바티가 시스템 문구로 축소한다. |
| 마왕 ↔ 레온 | 악의 허세 vs 정의의 허세 | 둘 다 대사를 연습해 왔는데 실전에서 꼬인다. |
| 마왕 ↔ 골딘 | 공포 연출 vs 유지비 | 멋진 연출은 거의 항상 비용으로 환산된다. |
| 마왕 ↔ 니아 | 보물 수호자 vs 보안 평가자 | 니아가 마왕성을 털면서 별점을 준다. |
| 바티 ↔ 골딘 | 운영 안정 vs 비용 통제 | 바티는 필요한 시스템을 말하고, 골딘은 예산을 깎는다. |
| 레온 ↔ 셀렌 | 열혈 정의 vs 절차 정의 | 레온은 외치고, 셀렌은 체크리스트를 꺼낸다. |
| 마왕 ↔ 부하 몬스터 | 거대한 계획 vs 단순 실행 | 마왕의 웅장한 명령을 몬스터가 문자 그대로 해석한다. |

### 1.3 대사 금지 규칙

- 직접적 고어, 잔혹한 학살 개그는 쓰지 않는다.
- 성인 요소를 쓰지 않는다.
- 기존 게임의 캐릭터명, UI명, 고유 이벤트를 연상시키는 문장을 쓰지 않는다.
- 인터넷 밈, 유행어에 의존하지 않는다. 시간이 지나도 쓸 수 있는 상황 개그로 쓴다.
- 캐릭터가 자기 역할을 설명하느라 대사가 딱딱해지면 안 된다. 설명은 바티나 UI가 담당하고, 캐릭터는 상황에 반응한다.

### 1.4 대사 길이 기준

| 사용처 | 권장 길이 | 성격 |
|---|---:|---|
| HUD 전투 로그 | 15~35자 | 짧고 즉각적 |
| 선택 유닛 대사 | 10~30자 | 캐릭터성 우선 |
| 전투 전 컷신 | 2~5줄 | 상황 설명 + 개그 |
| 보스 등장 대사 | 4~8줄 | 캐릭터 관계 강조 |
| 결산 대사 | 2~5줄 | 결과, 보상, 다음 전개 암시 |
| 엔딩 대사 | 5~10줄 | 감정선 우선, 개그는 한 박자만 |

## 3.1 CHR_DARKLORD_PLAYER: F급 신입 마왕 `{{player_name}}`

### 기본 정의

| 항목 | 내용 |
|---|---|
| 소속 | 마왕성 |
| 역할 | 플레이어 아바타, 캠페인 주인공, 마왕성 운영자 |
| 한 줄 설명 | 위엄을 갖추고 싶지만 운영 현실에 계속 발목 잡히는 F급 신입 마왕 |
| 캐릭터 원형 | 허세형 초보 관리자 + 성장형 지휘관 |
| 코미디 기능 | 장르적 허세를 제공하는 1타자 |
| 감정 핵심 | 인정 욕구, 책임감, 들키기 싫은 불안 |
| 플레이어 감정 연결 | “내가 약하지만 내 성과 부하는 내가 지킨다.” |

### 캐릭터 핵심

마왕은 바보가 아니다.
다만 “마왕답게 보이는 법”을 책과 상상으로만 배운 초보다.

초반의 마왕은 침입자 앞에서 대사를 멋지게 외우려 하지만, 왕좌 HP, 보물 손실, 함정 쿨타임 같은 현실 때문에 매번 흐름이 깨진다. 그래서 웃기다.
중반의 마왕은 자기 허세가 부하들에게 사기를 주는 방식임을 깨닫는다.
후반의 마왕은 더 이상 남의 마왕 흉내를 내지 않고, “초라한 동굴을 함께 키운 지휘관”으로 선다.

### 말투 DNA

| 항목 | 기준 |
|---|---|
| 기본 말투 | 거창한 선언형. “나의”, “이 성”, “공포”, “왕좌” 같은 단어를 좋아한다. |
| 무너질 때 | 갑자기 현실적이고 소심해진다. “그건 좀”, “예산 안에서”, “청소는 누가” 같은 말을 한다. |
| 전투 명령 | 짧고 분명하게. “입구를 막아라!”, “함정으로 유도!”, “왕좌는 절대 못 내준다!” |
| 감정선 | 부하가 다치면 허세가 사라지고 진심이 나온다. |
| 금지 | 잔혹한 학살자, 냉혈 폭군, 장황한 다크 판타지 독백 |

### 반복 개그

| 반복 개그 | 사용 방식 |
|---|---|
| 연습한 대사 실패 | 마왕이 준비한 대사를 꺼내려는 순간, 바티가 경고를 띄우거나 적이 엉뚱한 질문을 한다. |
| 문패 집착 | “언젠가 정식 마왕성 문패를 달겠다”는 말을 반복한다. |
| 왕좌 집착 | 왕좌는 권위의 상징이지만 실제 게임에서는 HP 오브젝트라서 자꾸 맞는다. |
| 악명 마케팅 | 악명을 공포가 아니라 홍보/브랜딩처럼 오해한다. |

### 성장 단계

| 단계 | 전투 범위 | 상태 | 대사 톤 |
|---|---:|---|---|
| 초반 | DAY 01~05 | 마왕 흉내를 내는 초보 | 허세가 많고 자주 당황한다. |
| 성장기 | DAY 06~15 | 운영을 배우는 지휘관 | 허세 뒤에 실제 명령이 붙는다. |
| 인정기 | DAY 16~25 | 악명을 관리하는 마왕 | 부하와 성을 우선한다. 농담도 여유가 있다. |
| 완성 | DAY 26~30 | 자기 방식의 진짜 마왕 | 말수가 줄고, 필요한 순간만 크게 선언한다. |

### 관계 훅

| 대상 | 관계 | 대사 방향 |
|---|---|---|
| 바티 | 가장 중요한 참모. 마왕의 허세를 깎지만, 마왕성을 실제로 굴러가게 한다. | 마왕이 선언하고 바티가 현실로 번역한다. |
| 레온 | 거울형 라이벌. 둘 다 초보로 시작해 각자의 역할을 배운다. | 서로 대사를 준비해 오고 서로 타이밍을 망친다. |
| 골딘 | 재무 압박. 마왕의 낭만을 숫자로 깎는다. | “위엄”과 “비용”의 충돌. |
| 니아 | 보물 보관실을 노리는 악우. | 체면을 지키려는 마왕과 별점 매기는 니아. |
| 푸딩 | 첫 부하. 보호 본능을 건드린다. | 푸딩이 단순하게 핵심을 찌른다. |
| 곱 | 돌격성 부하. 마왕이 전략을 배우게 만든다. | “때릴까?” “아직!” 반복. |
| 핀 | 허세 동지. 둘이 있으면 연출이 과해진다. | 바티가 둘 다 줄인다. |
| 셀렌 | 인간 측 검사관. 마왕성을 공식 위협으로 인정한다. | 마왕의 위엄을 등급표로 환산한다. |

### 대표 대사


## 3.2 CHR_BATI: 비서 박쥐 바티

### 기본 정의

| 항목 | 내용 |
|---|---|
| 소속 | 마왕성 |
| 역할 | 핵심 참모, 튜토리얼, HUD 경고, 결산, 작전 브리핑 |
| 한 줄 설명 | 마왕의 허세를 현실로 번역하는 건조한 박쥐 참모 |
| 캐릭터 원형 | 냉정한 비서 + 시스템 관리자 + 숨은 보호자 |
| 코미디 기능 | 마왕의 과장을 즉시 실무 문장으로 축소하는 2타자 |
| 감정 핵심 | 책임감, 안정 지향, 작지만 확실한 애정 |
| 플레이어 감정 연결 | “이 성은 허세만으로는 안 굴러간다. 그래서 내가 있다.” |

### 캐릭터 핵심

바티는 단순한 설명 NPC가 아니다.
마왕성이 실제로 돌아가게 만드는 사람이다.

바티의 대사는 대부분 시스템 정보처럼 시작하지만, 문장 끝에 미묘한 농담이나 애정이 들어간다.
마왕의 허세를 끊어내는 역할이지만, 마왕을 무시하지 않는다. 오히려 마왕이 성장할 때 가장 먼저 눈치채고, 가장 늦게 칭찬한다.

### 말투 DNA

| 항목 | 기준 |
|---|---|
| 기본 말투 | 정중한 존댓말. 업무 보고체. 짧고 정확하다. |
| 농담 방식 | 웃기려고 하지 않고 너무 건조해서 웃긴다. |
| 마왕 호칭 | 항상 “마왕님”을 쓴다. 화가 나도 호칭은 유지한다. |
| 감정 표현 | 직접 칭찬하지 않고 “효율적이었습니다”, “이번 판단은 나쁘지 않았습니다”처럼 말한다. |
| 금지 | 지나친 독설, 마왕을 무능한 바보로 취급하는 말투 |

### 반복 개그

| 반복 개그 | 사용 방식 |
|---|---|
| 시스템상 불가 | 마왕의 감성적 요청을 기능/예산/쿨타임 문제로 자른다. |
| 공포 수치 평가 | 협박이나 연설을 수치화한다. |
| 과한 연출 금지 | 마왕과 핀의 허세를 줄인다. |

| 성을 키우며 성장 | 패배를 복기하며 성장 |
| 최종적으로 진짜 마왕이 됨 | 최종적으로 진짜 용사가 됨 |

레온의 코미디는 초반에 강하고, 후반에는 줄인다.

## 6.1 마왕 ↔ 바티: 핵심 운영 콤비

### 관계 정의

| 항목 | 내용 |
|---|---|
| 관계명 | 허세와 실무의 계약 관계 |
| 핵심 정서 | 의존, 교정, 신뢰 |
| 코미디 원리 | 마왕이 장르적으로 말하면 바티가 시스템적으로 번역한다. |
| 대사 비율 | 마왕 1줄 → 바티 1줄 → 마왕 1줄. 가장 안정적인 3박자. |
| 사용처 | 튜토리얼, 방어 준비, 전투 경고, 결산, 패배 안내, 최종전 직전 |

### 대사 공식

```text
마왕: 거창한 선언 또는 감성적 요청
바티: 불가능/위험/비효율을 짧게 보고
마왕: 체면을 살리며 실행 가능한 명령으로 바꿈
```

### 초반 예시

```text
BANTER_PLAYER_BATI_EARLY_001
PLAYER: 이 동굴에 공포의 시대를 열겠다!
BATI: 현재 보유 방은 입구, 복도, 왕좌의 방입니다. 시대보다는 임대 공간에 가깝습니다.
PLAYER: 그럼 공포의 임대부터 시작한다!
```

```text
BANTER_PLAYER_BATI_EARLY_002
PLAYER: 침입자에게 마왕의 위엄을 보여줘라!
BATI: 위엄보다 입구 봉쇄가 먼저입니다. 위엄은 적이 멈춰 있을 때 효과가 큽니다.
PLAYER: 푸딩! 위엄 있게 막아라!
```

### 중반 예시

```text
BANTER_PLAYER_BATI_MID_001
PLAYER: 오늘 원정은 왕국을 떨게 할 첫걸음이다.
BATI: 실제 목표는 보급 상자에 마왕성 표식을 남기고 안전히 귀환하는 것입니다.
PLAYER: 왕국을 아주 작게 떨게 할 첫걸음이다.
```

```text
BANTER_PLAYER_BATI_MID_002
PLAYER: 바티, 나도 이제 좀 마왕다워졌나?
```

```text
BANTER_PLAYER_BATI_LATE_001
PLAYER: 왕국이 우리를 성채로 인정했다.
```

### 작성 주의

바티가 마왕을 지나치게 깎아내리면 관계가 불쾌해진다.
바티의 핵심은 “무시”가 아니라 “정리”다.
따라서 바티는 마왕의 허세를 현실에 맞게 줄이지만, 중요한 순간에는 가장 먼저 마왕을 지지해야 한다.

# 9. 대사 작성용 빠른 규칙표

| 캐릭터 | 첫 문장 기능 | 받아치는 인물 | 개그 결말 |
|---|---|---|---|
| 마왕 | 위엄 선언 | 바티/골딘/레온 | 현실에 맞춰 축소하거나 더 과장 |
| 바티 | 상황 보고 | 마왕 | 마왕이 체면 회복 시도 |
| 레온 | 정의 선언 | 마왕/셀렌 | 매뉴얼과 현실이 충돌 |
| 골딘 | 손실 경고 | 마왕/니아 | 뼈 농담 또는 예산 삭감 |
| 니아 | 보안 평가 | 마왕/골딘 | 별점/도난 농담 |
| 셀렌 | 점검/등급 | 마왕/레온 | 절차와 감정 충돌 |
| 푸딩 | 단순 진실 | 모두 | 말이 짧아서 오히려 정답 |
| 곱 | 돌격 제안 | 바티/마왕 | 기다리라는 명령과 충돌 |
| 핀 | 과장 연출 | 바티 | 주문명 축소 |

# 14. 완료 조건

이번 캐릭터 시트 v0.3의 완료 조건은 다음과 같다.

- `마왕-바티` 관계가 핵심 운영 콤비로 정의되어 있다.
- `마왕-레온` 관계가 핵심 성장형 라이벌 콤비로 정의되어 있다.
- 바티가 단순 튜토리얼 NPC가 아니라, 마왕의 코미디와 성장을 완성하는 참모로 설정되어 있다.
- 레온이 초반에는 코믹하지만 후반에는 진짜 라이벌로 성장하도록 단계별 말투가 정의되어 있다.
- 골딘, 니아, 셀렌이 각각 경제/보안/절차 축의 관계형 개그를 담당한다.
- DAY 01~03 데모 구간에 바로 넣을 수 있는 대사 샘플이 있다.
- 정규 캠페인 핵심 장면인 DAY 04, DAY 15, DAY 25, DAY 30의 대사 방향이 있다.
- JSON 데이터 구조로 캐릭터, 관계, 대사를 분리할 수 있다.
- 직접적 고어, 성인 요소, 기존 게임 복제 요소가 없다.

# 세션 종료 정리

## 1. 구현 대상

- `CHR_DARKLORD_PLAYER`, `CHR_BATI`, `CHR_GOLDIN`, `CHR_PUDDING`, `CHR_GOB`, `CHR_PYNN`, `CHR_EXPLORER_MILO`, `CHR_THIEF_NIA`, `CHR_HERO_LEON`을 데모 핵심 캐릭터로 확정한다.
- `CHR_SELEN`, `CHR_ROMAN`, `CHR_KALT`, `CHR_ILIA`, `CHR_SCRIBE_MARE`를 정규 캠페인 핵심 캐릭터로 확정한다.
- 마왕-바티, 마왕-레온, 마왕-골딘, 마왕-니아, 바티-골딘, 레온-셀렌 관계를 대사 생성의 핵심 관계로 확정한다.
- DAY 01~03 데모 대사는 v0.3 케미 기준으로 교체한다.

## 2. 데이터 구조

- `characters.json`: 캐릭터 ID, 표시명, 소속, 역할 태그, 초상화 키, 감정 태그, 말투 규칙, 금지 톤.
- `character_relationships.json`: 관계 ID, 포함 캐릭터, 우선순위, 코미디 패턴, 캠페인 단계별 변화.
- `dialogue_lines.json`: 대사 ID, 전투 ID, 단계, 화자, 대상, 감정 태그, 본문, 다음 대사 ID, 표시 조건.
- 추후 Godot에서는 `DataRegistry` 또는 별도 `DialogueManager`가 JSON을 읽고, `SignalBus`로 HUD/컷신/전투 로그에 전달한다.

## 3. UI에 표시할 정보

- 대화창: 캐릭터명, 초상화, 감정 태그, 대사 본문, 다음 대사 진행 표시.
- HUD 전투 로그: 짧은 버전의 핵심 대사, 경고, 방 지침 반응.
- 선택 유닛 정보: 몬스터 역할, 한 줄 성격, 짧은 선택 대사.
- 보스 등장 UI: 보스명, 역할 태그, 한 줄 성격, 현재 캠페인 단계.
- 결산 화면: 바티의 작전 평가, 골딘의 손익 평가, 마왕의 승리/패배 반응.
