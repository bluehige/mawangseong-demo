# 04. 배경, 빈 공간, 미연결 상태 규칙

## 1. 사용자 아이디어에 대한 판단

사용자 제안:

> 배경 이미지를 미리 만들어놓고 그리드 위에 원하는 오브젝트를 설치하면 빈 공간 문제를 해결할 수 있다.

이 판단은 **절반은 맞고, 절반은 부족하다.**

### 맞는 부분

- 맵 바깥쪽 암석/어둠/지반을 배경 한 장으로 처리하면 비어 보이지 않는다.
- 카메라 이동 시 모자라는 공간을 자연스럽게 채울 수 있다.
- 쿼터뷰 던전의 분위기를 안정적으로 유지할 수 있다.

### 부족한 부분

- 통로가 연결되기 전과 후의 시각 변화는 배경만으로 표현할 수 없다.
- 닫힌 통로 끝, 막힌 문, 가짜 확장 슬롯은 별도 오브젝트나 edge 규칙이 필요하다.
- floor cell의 exposed edge는 주변 연결 여부에 따라 바뀌어야 한다.

따라서 최종 정답은:

> **배경 이미지를 깔고, 그 위에 floor/edge/socket cap/doorway를 조립한다.**

---

## 2. 빈 공간 처리 규칙

빈 공간은 최소 3종으로 나눈다.

| 상태 | 의미 | 시각 표현 |
|---|---|---|
| void | 아직 마왕성이 차지하지 않은 외부 영역 | 어둠, 암석, 심연 배경 |
| rock | 활성 영역 안이지만 아직 파내지 않은 암반 | 막힌 흙벽, 굴착 전 암반 |
| open_placeholder | 추후 확장 가능한 건설 슬롯 | 흐릿한 건설 가이드, 마법 윤곽 |

즉, 빈 공간이 모두 같은 "검은색 빈칸"이 아니다.

---

## 3. 미연결 소켓 시각 규칙

소켓이 열리지 않았을 때는 다음 셋 중 하나를 사용한다.

### closed socket
- 동굴벽으로 막힘
- 철문이 닫힘
- 벽돌로 봉인됨

### open_placeholder socket
- 연결 예정임을 암시하는 문틀
- 마법 윤곽선
- 건설 슬롯 표식

### connected socket
- 통로와 자연스럽게 이어지는 문틀
- 바닥이 연결되어 있고, wall cap이 제거됨

---

## 4. 연결 전후 변화 규칙

예를 들어 방의 동쪽 소켓이 `closed` 상태라면:

- 방 동쪽 바닥 edge는 비연결 타일
- 동쪽 벽/문틀은 막힌 상태 이미지
- walkable은 false

동쪽 소켓이 `connected` 상태가 되면:

- 해당 위치 floor mask 재계산
- 동쪽 문틀을 open doorway sprite로 교체
- corridor blueprint를 추가 배치
- walkable true로 전환

즉, 연결은 단순히 corridor를 추가하는 것이 아니라,
**기존 방 가장자리의 타일과 벽 오브젝트도 함께 바뀌어야 한다.**

## 5. exposed edge와 background의 관계

노출 가장자리는 배경과 별개다.

예시:
- floor tile 하나가 단독으로 놓였으면 mask 0 이미지가 필요함
- 이때 floor 아래/밖은 background가 보이되,
  floor 가장자리에는 edge/skirt shadow가 있어야 한다

즉:

- BackgroundVoidLayer = 큰 배경
- FloorLayer = 바닥 다이아몬드
- EdgeSkirtLayer = 바닥이 끊기는 지점의 경사/그림자

이 세 층이 같이 있어야 자연스럽다.
