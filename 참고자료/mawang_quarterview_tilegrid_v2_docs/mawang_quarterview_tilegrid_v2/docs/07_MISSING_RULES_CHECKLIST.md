# 07. 빠지기 쉬운 규칙 체크리스트

사용자가 빼먹었을 수 있고, Codex도 놓치기 쉬운 규칙들이다.

## 1. 이동 가능 여부와 시각 표현은 분리한다

- 예쁜 바닥처럼 보여도 walkable false일 수 있다.
- 반대로 배경에 그림이 있어도 cell이 없으면 이동 금지다.

## 2. 큰 오브젝트는 1장으로 끝내지 않는다

예:
- 왕좌
- 커다란 보물더미
- 큰 기둥
- 병영 침상 세트

가능하면 front/back로 나눠야 유닛이 앞뒤로 지나갈 수 있다.

## 3. 방 내부 소품은 floor mask를 바꾸지 않는다

소품을 올린다고 floor 연결 자체가 변하면 안 된다.
필요 시 `object_block`로 이동만 막는다.

## 4. 통로 길이 1칸, 2칸, 3칸, 코너, T자, 십자형 블루프린트를 따로 가진다

floor mask만으로 전체 던전 플레이 감각이 나오지 않는다.
통로 구조는 blueprint 레벨에서도 다양성이 필요하다.

## 5. 건설 가능한 곳과 연결 가능한 곳을 구분한다

- buildable = 방을 올릴 수 있는 셀
- connectable = 소켓이 있는 방향

둘은 다를 수 있다.

## 6. camera bounds도 grade에 따라 확장한다

grid가 넓어졌는데 카메라가 그대로면 잘린다.

## 7. minimap도 logical grid를 기준으로 만든다

미니맵은 렌더 이미지 캡처가 아니라 logical cell data를 직접 그린다.

## 8. 저장 데이터에는 theme와 socket state를 꼭 저장한다

등급이 바뀌고 로드했을 때 연결 상태가 유지돼야 한다.

## 9. 디버그 키를 반드시 둔다

- F3: active cell 보기
- F4: walkable 보기
- F5: floor mask 번호 보기
- F6: socket 상태 보기
- F7: room id 보기

Codex가 이 디버그를 안 넣으면 규칙이 맞게 돌아가는지 확인하기 어렵다.
