# 02. 마왕성 업그레이드와 그리드 확장 규칙

## 1. 구현 권장 방식

### 권장안: 고정 최대 그리드 + 등급별 활성 구역 확장

예시:

- 전체 최대 그리드: `20 x 20`
- 시작 활성 영역(F급): 중심부 `8 x 8`
- E급: `10 x 10`
- D급: `12 x 12`
- C급: `14 x 14`
- B급: `16 x 16`
- A급: `18 x 18`
- S급: `20 x 20`

이 방법의 장점:

1. 좌표 재배치가 필요 없음
2. 저장 데이터 구조가 안정적임
3. 카메라 범위 조정이 쉬움
4. 기존 방을 옮기지 않아도 됨
5. 건설 가능한 슬롯만 늘어나므로 구현이 간단함

---

## 2. 활성 구역 정의 방식

활성 구역은 다음 둘 중 하나로 구현할 수 있다.

### 방식 A. 직사각형 활성 영역

```json
{
  "grade": "F",
  "active_rect": {"x": 6, "y": 6, "w": 8, "h": 8}
}
```

장점:
- 가장 단순함
- Codex 구현 쉬움

단점:
- 성장 모양이 기계적으로 보일 수 있음

### 방식 B. unlock mask 방식 (권장)

```json
{
  "grade": "F",
  "unlock_cells": [[8,8],[8,9],[9,8],[9,9], ... ]
}
```

장점:
- 동굴처럼 비정형 성장 가능
- 특정 방향 확장 가능
- 이벤트나 보스방 해금과 연동 쉬움

단점:
- 데이터가 조금 많음

정규버전은 **unlock mask 방식**이 더 적합하다.

## 3. 마왕성 등급별 권장 그리드 규모

| 등급 | 활성 셀 규모(권장) | 체감 역할 |
|---|---:|---|
| F | 8x8 수준 | 초라한 동굴 |
| E | 10x10 수준 | 수상한 소굴 |
| D | 12x12 수준 | 지하 요새 |
| C | 14x14 수준 | 흑철 성채 |
| B | 16x16 수준 | 마력 왕성 |
| A | 18x18 수준 | 정식 마왕성 |
| S | 20x20 수준 | 대마왕성 |

여기서 중요한 것은 **실제 파일 수가 늘어나는 것이 아니라, 같은 타일을 더 넓은 범위에 반복 사용한다는 점**이다.

---

## 4. 확장 시 추가로 바뀌는 것

마왕성 업그레이드는 단순히 active cell만 늘어나면 밋밋하다.
따라서 아래 요소도 같이 바꾼다.

1. 테마 팔레트
2. 벽 장식
3. 바닥 데칼 빈도
4. 문/기둥 디자인
5. 배경 이미지 테마
6. 허용 방 종류

예시:

- F급: 흙 동굴, 뼈 장식 적음
- D급: 흑철 문, 보라 횃불, 제단
- A급: 왕좌 장식, 마력 문양, 마계 석재

---

## 5. 확장 순서 규칙

활성 구역이 늘어날 때 무조건 모든 바깥 셀이 한꺼번에 열리면 안 된다.
다음 중 하나를 선택한다.

### 권장 규칙 1. 링 확장

- 현재 활성 영역 바깥 한 줄이 새로 열린다
- 가장 직관적

### 권장 규칙 2. 방향 확장

- 특정 이벤트/등급에서 북쪽 날개, 동쪽 날개처럼 부분 확장
- 더 전략적으로 보임

### 권장 규칙 3. 핵심 방 조건 확장

- 병영 업그레이드 시 병영 구역 인접 셀 확장
- 마력로 해금 시 심부 영역 확장

정규버전 1.0은 **링 확장 + 일부 방향 확장 이벤트** 조합이 가장 안전하다.
