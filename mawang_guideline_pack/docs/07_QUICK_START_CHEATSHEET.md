# 07. 빠른 시작 치트시트

새 세션을 시작할 때 이것만 붙여도 기본 방향을 잃지 않는다.

---

## 프로젝트 한 줄 요약

**「마왕님, 마왕성은 누가 지켜요?」는 F급 신입 마왕이 초라한 동굴을 마왕성으로 키우고, 슬라임·고블린·임프를 지휘해 탐험가·도적·견습 용사의 침공을 막는 2D 쿼터뷰 실시간 마왕성 방어 데모다.**

---

## 고정 기준

- Godot 4.5.
- GDScript.
- PC 16:9, 1920×1080.
- 2D 쿼터뷰.
- 큐트 호러 판타지.
- 3일짜리 기능 테스트 데모.
- 슬롯형 마왕성, 추후 자유건설 확장 가능.
- 몬스터 3종: 슬라임, 고블린, 임프.
- 적 3종: 탐험가, 도적, 견습 용사.
- 전투: 자동 전투 + 전체 지침 + 방 지침 + 직접 조종.

---

## 데모에서 반드시 보여줄 것

1. 입구에서 적이 들어온다.
2. 도적은 보물 보관실로 간다.
3. 견습 용사는 왕좌의 방으로 간다.
4. 슬라임은 길을 막는다.
5. 고블린은 근접 공격한다.
6. 임프는 화염구를 쏜다.
7. 사수/총공격/생존 우선 지침이 작동한다.
8. 방 지침으로 함정 유도/입구 봉쇄가 가능하다.
9. 선택 몬스터 1마리를 직접 조종할 수 있다.
10. 3일차 견습 용사를 격퇴하면 클리어된다.

---

## 가장 먼저 만들 순서

1. 프로젝트 초기화.
2. 쿼터뷰 방 구조.
3. 유닛 생성과 이동.
4. 자동 전투.
5. 행동지침.
6. 직접 조종.
7. 전투 UI.
8. 3일 루프.
9. 리소스 교체.
10. 밸런스.

---

## Codex에 바로 넣을 최소 명령

```text
Godot 4.5와 GDScript로 「마왕님, 마왕성은 누가 지켜요?」의 2D 쿼터뷰 기능 테스트 데모를 구현해줘.

범위는 3일 데모야. 마왕성은 입구, 가시 복도, 병영, 보물 보관실, 회복 둥지, 왕좌의 방으로 구성하고, 몬스터는 슬라임/고블린/임프, 적은 탐험가/도적/견습 용사만 넣어줘.

전투는 자동 전투 기반이고, 플레이어는 전체 지침(사수, 총공격, 생존 우선), 방 지침(입구 봉쇄, 함정 유도, 후퇴선 유지), 선택 몬스터 직접 조종으로 개입해야 해.

우선 Placeholder 그래픽으로 만들고, assets 폴더에 실제 PNG를 나중에 교체할 수 있게 구조화해줘.

먼저 프로젝트 구조와 Phase별 구현 계획을 제시한 뒤, Phase 0부터 구현해줘.
```

---

## 금지/보류

- 자유건설 전체 에디터는 데모 이후.
- 진화는 데모 이후.
- 특수 몬스터는 데모 이후.
- 장비는 데모 이후.
- 세력별 악명은 데모 이후.
- 성인 요소 없음.
- 직접적 고어 없음.
- 기존 게임 캐릭터/명칭/UI 복제 금지.
