# 04. 세션 로드맵과 산출물

이 문서는 제작 세션을 어떻게 나눌지 정리한 로드맵이다. 각 세션은 하나의 기능 묶음만 완성한다.

---

## 1. 전체 제작 순서

| 세션 | 이름 | 핵심 산출물 |
|---:|---|---|
| 0 | 프로젝트 초기화 | Godot 프로젝트, Autoload, 씬 전환 |
| 1 | 맵/방 구조 | 쿼터뷰 마왕성, RoomGraph |
| 2 | 유닛 기본 | 몬스터/적 생성, 이동 |
| 3 | 기본 전투 | 자동 공격, HP, 전투불능 |
| 4 | 행동지침 | 전체 지침, 방 지침 |
| 5 | 직접 조종 | 선택 몬스터 직접 이동/공격/스킬 |
| 6 | 전투 UI | TopBar, 방 목록, 선택 유닛, 명령 패널 |
| 7 | 관리 화면 | 건설/몬스터/방어 준비/다음 날 |
| 8 | 몬스터 관리 화면 | 몬스터 카드, 스킬 슬롯, 능력치 |
| 9 | 3일 웨이브 | 1~3일차 진행과 결산 |
| 10 | 그래픽 교체 구조 | SpriteFrames/asset 경로 정리 |
| 11 | 밸런스 | 수치 조정, 웨이브 난이도 조정 |
| 12 | 데모 패키징 | 실행 검수, README, 빌드 준비 |

---

## 2. 세션 0 — 프로젝트 초기화

### 시작 명령

```text
Godot 4.5 프로젝트를 초기화하고 「마왕님, 마왕성은 누가 지켜요?」의 2D 쿼터뷰 데모 기반 구조를 만들어줘.
Main.tscn, GameRoot.tscn, GameState, SignalBus, Constants, DataRegistry를 만들고 관리 화면과 전투 화면을 임시로 전환할 수 있게 해줘.
```

### 완료 조건

- 실행 시 오류 없음.
- GameRoot 표시.
- 임시 버튼으로 화면 전환 가능.
- 현재 일차/자원/마왕성 HP가 GameState에 저장됨.

---

## 3. 세션 1 — 맵/방 구조

### 시작 명령

```text
쿼터뷰 마왕성 맵과 방 구조를 구현해줘.
방은 입구, 가시 복도, 병영, 보물 보관실, 회복 둥지, 왕좌의 방, 건설 슬롯 1개로 구성해줘.
각 방은 RoomNode로 만들고 RoomGraph를 통해 방 사이 경로를 반환하게 해줘.
```

### 산출물

- DungeonMap.tscn.
- RoomNode.tscn.
- RoomGraph.gd.
- rooms.json.

### 완료 조건

- 방 클릭 가능.
- 방 이름 표시 가능.
- 입구 → 왕좌의 방 경로 반환.
- 입구 → 보물 보관실 경로 반환.

---

## 4. 세션 2 — 유닛 기본

### 시작 명령

```text
CharacterBody2D 기반 Unit 구조를 만들고 슬라임, 고블린, 임프, 탐험가, 도적, 견습 용사를 생성해줘.
각 유닛은 방 ID, 목표 방 ID, HP, ATK, DEF, 이동속도, 사거리, 공격 주기를 가져야 해.
적은 입구에서 등장해 목표 방으로 이동해야 해.
```

### 완료 조건

- 아군 3종 생성.
- 적 3종 생성.
- 적 목표 이동.
- 도적은 보물 보관실로 이동.
- 견습 용사는 왕좌의 방으로 이동.

---

## 5. 세션 3 — 기본 전투

### 시작 명령

```text
자동 전투를 구현해줘.
유닛은 사거리 안의 적을 탐색하고 공격해야 하며, 피해 공식은 max(1, ATK - DEF * 0.5)로 해줘.
임프는 원거리 화염구 투사체를 발사하고, HP가 0 이하가 되면 DOWN 상태로 전환해줘.
```

### 완료 조건

- 근접 공격 작동.
- 원거리 화염구 작동.
- HP 감소.
- 전투불능 처리.
- 전투 로그 기본 출력.

---

## 6. 세션 4 — 행동지침

### 시작 명령

```text
전체 지침과 방 지침을 구현해줘.
전체 지침은 사수, 총공격, 생존 우선이고, 방 지침은 입구 봉쇄, 함정 유도, 후퇴선 유지야.
지침이 UnitAI 행동 우선순위에 영향을 주도록 만들어줘.
```

### 완료 조건

- 지침 UI 버튼 작동.
- 사수 시 추격 제한.
- 총공격 시 적극 추격.
- 생존 우선 시 HP 35% 이하 후퇴.
- 함정 유도 시 가시 복도 쪽으로 전투가 유도됨.

---

## 7. 세션 5 — 직접 조종

### 시작 명령

```text
선택한 몬스터 1마리를 직접 조종할 수 있게 해줘.
좌클릭으로 유닛 선택, 직접 조종 버튼으로 모드 진입, 우클릭으로 이동/공격 지정, 스킬 버튼 또는 숫자키로 스킬 사용, 해제 시 AI 복귀를 구현해줘.
```

### 완료 조건

- 유닛 선택 링 표시.
- 직접 조종 버튼 작동.
- 우클릭 이동 가능.
- 임프 화염구 수동 사용 가능.
- 해제 후 자동 AI 복귀.

---

## 8. 세션 6 — 전투 UI

### 시작 명령

```text
전투 화면 HUD를 구현해줘.
상단 자원바, 좌측 미니맵/방 목록/전투 로그, 우측 선택 유닛 패널, 하단 전체 지침/방 지침/직접 조종 버튼을 만들어줘.
우선 기능 중심으로 만들고 디자인은 어두운 패널과 보라색 포인트만 적용해줘.
```

### 완료 조건

- 자원/날짜/마왕성 HP 표시.
- 방 목록 상태 표시.
- 선택 유닛 정보 표시.
- 행동지침 버튼 작동.
- 전투 로그 누적 표시.

---

## 9. 세션 7 — 관리 화면

### 시작 명령

```text
마왕성 관리 화면을 만들어줘.
중앙에 마왕성 방 배치를 보여주고, 하단에는 건설, 몬스터, 침공 작전, 방어 준비, 다음 날 버튼을 배치해줘.
데모에서는 건설은 제한적으로 표시하고, 방어 준비 버튼을 누르면 전투 화면으로 이동하게 해줘.
```

### 완료 조건

- 관리 화면 진입.
- 방 클릭 정보 표시.
- 방어 준비 → 전투 전환.
- 다음 날 진행 구조 연결.

---

## 10. 세션 8 — 몬스터 관리 화면

### 시작 명령

```text
몬스터 관리 화면을 만들어줘.
좌측에는 슬라임, 고블린, 임프 카드 목록을 보여주고, 우측에는 선택 몬스터의 능력치, 충성도, 스킬 슬롯 3칸, 훈련/배치/스킬 교체/돌아가기 버튼을 표시해줘.
데모에서는 훈련과 스킬 교체는 비활성 또는 간단 처리로 둬도 돼.
```

### 완료 조건

- 몬스터 목록 표시.
- 선택 몬스터 정보 표시.
- 스킬 슬롯 3칸 표시.
- 돌아가기 버튼 작동.

---

## 11. 세션 9 — 3일 웨이브와 결산

### 시작 명령

```text
3일짜리 데모 루프를 구현해줘.
1일차는 탐험가 2명, 2일차는 탐험가 2명과 도적 1명, 3일차는 탐험가 2명, 도적 1명, 견습 용사 1명이 등장하게 해줘.
전투 후 결산 화면에서 금화, 마력, 악명, 경험치, 피해 상황을 보여줘.
3일차 견습 용사를 격퇴하면 데모 클리어로 처리해줘.
```

### 완료 조건

- 1~3일차 진행.
- 결산 표시.
- 데모 클리어 표시.
- 패배 표시.

---

## 12. 세션 10 — 그래픽 교체 구조

### 시작 명령

```text
Placeholder 그래픽을 실제 이미지 리소스로 교체할 수 있도록 asset 경로와 SpriteFrames 구조를 정리해줘.
assets/sprites/monsters, enemies, rooms, tiles, effects, ui 폴더를 기준으로 유닛 ID와 리소스 파일을 연결할 수 있게 만들어줘.
```

### 완료 조건

- 기존 Placeholder 유지.
- 실제 PNG 파일을 넣으면 교체 가능.
- 유닛별 SpriteFrames 분리.
- 리소스 누락 시 Placeholder로 fallback.

---

## 13. 세션 11 — 밸런스

### 시작 명령

```text
현재 3일 데모의 전투 밸런스를 점검하고 수치를 조정해줘.
목표는 1일차는 쉽게 승리, 2일차는 지침을 쓰면 안정 승리, 3일차는 직접 조종과 임프 스킬을 써야 안정 승리하는 난이도야.
```

### 완료 조건

- 몬스터/적 수치 조정.
- 웨이브 간격 조정.
- 결과별 테스트 케이스 정리.

---

## 14. 세션 12 — 데모 패키징

### 시작 명령

```text
3일짜리 기능 데모를 패키징하기 전 최종 점검해줘.
실행 방법, 조작법, 알려진 제한사항, 다음 개발 항목을 README로 정리하고, 데모 완료 체크리스트를 통과하는지 확인해줘.
```

### 완료 조건

- README 작성.
- 조작법 정리.
- 알려진 버그/제한사항 정리.
- 최종 체크리스트 통과.
