프로젝트명은 「마왕님, 마왕성은 누가 지켜요?」다.

이 프로젝트는 F급 신입 마왕이 초라한 동굴을 진짜 마왕성으로 키우고, 부하 몬스터를 배치·육성·지휘해 탐험가와 용사의 침공을 막는 2D 쿼터뷰 실시간 마왕성 방어 게임이다.

고정 기준:
- 엔진은 Godot 4.5, 언어는 GDScript다.
- 화면은 PC 16:9, 1920×1080 기준이다.
- 전투 시점은 2D 쿼터뷰다.
- 그래픽 톤은 큐트 호러 판타지다. 어두운 동굴, 보라색 마력 조명, 귀여운 몬스터, 작은 해골 장식이 핵심이다.
- 첫 데모는 3일짜리 기능 테스트 데모다.
- 마왕성은 데모에서 슬롯형 방 구조로 제작하되, 추후 자유건설로 확장 가능하게 데이터 구조를 만든다.
- 초기 몬스터는 슬라임, 고블린, 임프 3종이다.
- 초기 적은 탐험가, 도적, 견습 용사 3종이다.
- 전투는 자동 전투를 기본으로 하며, 플레이어는 전체 지침, 방 지침, 선택 몬스터 직접 조종으로 개입한다.
- 전체 지침은 사수, 총공격, 생존 우선이다.
- 방 지침은 입구 봉쇄, 함정 유도, 후퇴선 유지다.
- 직접 조종은 선택 몬스터 1마리만 가능하다.
- 데모의 승리 조건은 3일차 견습 용사 격퇴다.
- 패배 조건은 왕좌의 방 HP 0이다.

데모에 반드시 들어갈 것:
1. 관리 화면 → 방어 준비 → 전투 → 결산 → 다음 날 루프.
2. 입구, 가시 복도, 병영, 보물 보관실, 회복 둥지, 왕좌의 방.
3. 슬라임은 탱커로 길을 막는다.
4. 고블린은 근접 딜러로 빠르게 공격한다.
5. 임프는 원거리 마법 딜러로 화염구를 쏜다.
6. 탐험가는 기본 침입자다.
7. 도적은 보물 보관실을 노린다.
8. 견습 용사는 왕좌의 방을 노리는 미니 보스다.
9. 가시 복도 함정은 피해와 둔화를 준다.
10. 전투 로그와 선택 유닛 정보가 표시된다.

데모에서 제외할 것:
- 자유건설 전체 에디터.
- 다층 마왕성.
- 진화.
- 특수 몬스터.
- 장비.
- 세력별 악명.
- 대규모 스토리.
- 성인 요소.
- 직접적 고어.
- 기존 게임의 캐릭터, UI, 명칭, 이벤트 복제.

기술 구조:
- CharacterBody2D 기반 유닛.
- TileMapLayer 기반 맵.
- CanvasLayer 기반 HUD.
- JSON 기반 데이터 확장 가능 구조.
- Autoload는 GameState, SignalBus, 필요 시 DataRegistry를 사용한다.
- Placeholder 그래픽으로 먼저 구현하고, assets 폴더의 PNG 리소스로 나중에 교체한다.

작업 원칙:
1. 한 세션에는 하나의 기능 묶음만 다룬다.
2. 먼저 실행 가능한 최소 구현을 만든다.
3. 새 아이디어는 데모 핵심, 데모 보조, 확장판 후보로 분류한다.
4. 데모 핵심 범위 밖의 기능은 TODO로 남기고 구현하지 않는다.
5. 산출물은 씬, 스크립트, 데이터, 완료 조건 단위로 정리한다.
6. 모든 중요한 변경은 결정 로그에 기록한다.
