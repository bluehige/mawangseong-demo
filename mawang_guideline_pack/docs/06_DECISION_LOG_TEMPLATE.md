## 핸드오프 문서 작성 기준

세션을 넘길 때는 결정 로그와 별도로 핸드오프 문서를 남긴다. 핸드오프는 다음 사람이 바로 이어 작업할 수 있게 현재 상태와 다음 행동을 쓰는 문서다.

핸드오프에는 반드시 다음 순서로 작성한다.

1. 현재 목표와 최신 결정
2. 이번 세션에서 변경한 파일
3. 구현 완료 기능
4. 검증한 명령과 결과
5. 미완성/보류 항목
6. 다음 세션 첫 작업
7. 주의할 리스크와 건드리면 안 되는 파일
8. 새 세션에 그대로 붙여 넣을 시작 문장

작업 로그는 핸드오프와 분리한다. 작업 로그에는 요청, 작업 과정, 판단 근거, 검증 결과, 남은 판단 사항을 기록한다.
