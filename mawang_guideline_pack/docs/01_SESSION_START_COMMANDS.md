# 01. 세션 시작 명령문 모음

이 문서는 새 ChatGPT 세션, Codex 세션, 이미지 생성 세션, QA 세션을 시작할 때 붙여 넣을 명령문 모음이다. 필요한 항목을 그대로 복사해 사용한다.

---

## 1. 모든 세션 공통 시작 명령

```text
프로젝트명은 「마왕님, 마왕성은 누가 지켜요?」다.

핵심 방향:
- 신입 마왕이 초라한 동굴을 진짜 마왕성으로 키우는 게임.
- 장르: 마왕성 건설/운영 + 몬스터 육성 + 실시간 전투 디펜스.
- 엔진: Godot 4.5, GDScript.
- 화면: PC 16:9, 1920×1080 기준.
- 시점: 2D 쿼터뷰.
- 그래픽 톤: 큐트 호러 판타지.
- 데모: 3일짜리 기능 구현 테스트.
- 초기 몬스터: 슬라임, 고블린, 임프.
- 초기 적: 탐험가, 도적, 견습 용사.
- 마왕성 구조: 데모는 슬롯형 방 구조, 추후 자유건설 확장 가능하게 설계.
- 전투: 자동 전투 + 전체 지침 + 방 지침 + 선택 몬스터 직접 조종.

이번 세션에서 해야 할 일:
[여기에 이번 세션 목표 입력]

반드시 지킬 것:
1. 데모 핵심 범위를 넘는 기능은 제안만 하고 구현/상세화는 보류한다.
2. 기능은 Godot 4.5와 GDScript로 구현 가능한 수준으로 구체화한다.
3. Placeholder 그래픽으로 먼저 구현하고, 리소스 교체는 후순위로 둔다.
4. 원작의 캐릭터, UI, 명칭, 이벤트를 복제하지 않는다.
5. 성인 요소와 직접적 고어는 넣지 않는다.
6. 산출물은 파일/씬/스크립트/데이터/완료 조건 단위로 정리한다.

출력 형식:
- 목표
- 구현/기획 범위
- 필요한 파일 또는 데이터
- 단계별 작업
- 완료 조건
- 다음 세션으로 넘길 사항
```

## 2. 기획 세션 시작 명령

```text
이번 세션은 기획 세션이다.

목표:
[예: 행동지침 시스템 v0.1을 데모 구현 가능한 수준으로 확정]

출력은 다음 형식으로 작성한다.
1. 이번 세션에서 확정할 범위
2. 데모에 반드시 들어갈 기능
3. 데모에서 제외할 기능
4. 데이터 구조
5. UI 표시 방식
6. 구현 우선순위
7. 완료 조건
8. Codex에 넘길 제작 지시문

주의:
- 아이디어를 넓히지 말고 구현 가능한 규칙으로 닫는다.
- 확장 아이디어는 별도 ‘확장 후보’ 항목으로만 정리한다.
- 기존 확정 기준인 쿼터뷰, 3일 데모, 몬스터 3종, 적 3종을 바꾸지 않는다.
```

## 3. Codex 구현 세션 시작 명령

```text
이번 세션은 Godot 4.5 / GDScript 구현 세션이다.

프로젝트는 「마왕님, 마왕성은 누가 지켜요?」이며, 2D 쿼터뷰 마왕성 방어 데모를 만든다.

이번 구현 목표:
[예: Phase 2 유닛 생성과 이동 구현]

작업 규칙:
1. 먼저 현재 프로젝트 구조를 확인한다.
2. 기존 파일을 무리하게 갈아엎지 말고 필요한 파일만 수정한다.
3. 실행 가능한 최소 기능을 우선 구현한다.
4. Placeholder 그래픽을 사용하되, 나중에 실제 PNG 리소스로 교체할 수 있게 SpriteFrames/Texture 경로를 분리한다.
5. 데이터는 추후 JSON으로 분리할 수 있도록 ID 기반으로 작성한다.
6. 완료 후 테스트 절차와 알려진 제한사항을 적는다.

출력 형식:
- 변경/생성 파일 목록
- 핵심 구현 내용
- 실행 방법
- 테스트 방법
- 다음 작업 제안 1개
```

## 4. UI 설계 세션 시작 명령

```text
이번 세션은 UI 설계 세션이다.

목표:
[예: 전투 화면 HUD를 Godot Control 노드 기준으로 세분화]

기준:
- PC 16:9, 1920×1080.
- 중앙은 쿼터뷰 전투 맵.
- 상단은 자원/날짜/마왕성 체력.
- 좌측은 미니맵, 방 목록, 전투 로그.
- 우측은 선택 유닛 정보.
- 하단은 전체 지침, 방 지침, 직접 조종 버튼.
- 큐트 호러 톤. 어두운 배경, 보라색 포인트, 읽기 쉬운 글자.

출력 형식:
1. 화면 구성 표
2. 노드 트리
3. 각 패널의 필수 정보
4. 버튼 상태
5. 반응형/해상도 기준
6. 구현 우선순위
7. Codex용 구현 지시문
```

## 5. 이미지 리소스 생성 세션 시작 명령

```text
이번 세션은 gpt-image-2 이미지 리소스 제작 세션이다.

목표:
[예: 슬라임, 고블린, 임프의 쿼터뷰 SD 캐릭터 기준 이미지 제작]

스타일 기준:
- 큐트 호러 판타지.
- 쿼터뷰 게임 스프라이트.
- 어두운 동굴 마왕성 분위기.
- 보라색 마력 조명.
- 귀엽고 읽기 쉬운 실루엣.
- 투명 배경 PNG로 후처리 가능해야 함.
- UI 텍스트는 이미지에 넣지 않음.

출력 형식:
1. 리소스 목록
2. 파일명 규칙
3. 이미지 생성 프롬프트
4. 후처리 지침
5. Godot 적용 위치
6. 품질 체크 기준
```

## 6. 전투 밸런스 세션 시작 명령

```text
이번 세션은 전투 밸런스 세션이다.

목표:
[예: 1~3일차 웨이브 난이도와 몬스터 수치를 조정]

기준:
- 1일차는 거의 이겨야 한다.
- 2일차는 방 지침을 쓰면 안정적으로 이겨야 한다.
- 3일차는 직접 조종과 임프 스킬을 써야 안정적으로 이겨야 한다.
- 패배해도 원인을 이해할 수 있어야 한다.

출력 형식:
1. 현재 수치표
2. 문제 진단
3. 조정안
4. 기대 결과
5. 테스트 케이스
6. JSON 또는 GDScript 데이터 변경안
```

## 7. QA/버그 수정 세션 시작 명령

```text
이번 세션은 QA/버그 수정 세션이다.

목표:
[예: 적이 목표 방으로 이동하지 않는 문제 수정]

작업 규칙:
1. 재현 절차를 먼저 정리한다.
2. 원인을 추정한 뒤 관련 파일을 찾는다.
3. 최소 변경으로 수정한다.
4. 수정 후 테스트 절차를 작성한다.
5. 수정 과정에서 새 기능을 추가하지 않는다.

출력 형식:
- 버그 요약
- 재현 절차
- 원인
- 수정 파일
- 수정 내용
- 테스트 결과
- 남은 위험
```

## 8. 세션 종료 요약 명령

```text
이번 세션 결과를 다음 형식으로 요약해줘.

1. 세션 목표
2. 확정된 결정
3. 생성/수정한 파일
4. 구현 완료 기능
5. 검증한 명령과 결과
6. 아직 미완성인 기능
7. 다음 세션에서 먼저 해야 할 일
8. 변경된 리스크
9. 결정 로그에 남길 문장
10. Codex에 그대로 넘길 시작 지시문

요약은 다음 세션에 그대로 붙여 넣을 수 있게 작성해줘.
```

## 9. Codex 핸드오프 문서 작성 명령

```text
이번 세션을 넘기기 전에 핸드오프 문서를 업데이트해줘.

핸드오프에는 다음을 반드시 포함해줘.

1. 현재 목표와 최신 결정
2. 이번 세션에서 실제로 바꾼 파일
3. 구현된 기능과 아직 미완성인 기능
4. 실행/테스트 명령과 결과
5. 다음 세션에서 가장 먼저 해야 할 일
6. 주의할 리스크와 건드리면 안 되는 파일/변경
7. 새 세션에 그대로 붙여 넣을 Codex 시작 문장

핸드오프와 별도로 작업 로그도 남겨줘.
작업 로그에는 요청, 작업 내용, 검증 결과, 남은 판단 사항을 기록해줘.
```
