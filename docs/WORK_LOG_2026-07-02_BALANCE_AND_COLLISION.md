# 작업 로그: 유닛 충돌과 전투 밸런스 1차 조정

작성일: 2026-07-02

## 요청

- `mawang_guideline_pack/docs/05_QA_BALANCE_CHECKLIST.md`를 기준으로 전투 밸런스 계획을 세우고 진행.
- 도둑이 유닛을 통과하지 못하고, 유닛을 만나면 돌아서 가도록 유닛 간 충돌 체크를 확인하고 미구현 시 구현.
- 근접 전투가 붙어서 진행되어야 하므로 콜라이더는 너무 크게 잡지 않음.

## 밸런스 판단 기준

QA 체크리스트의 난이도 목표를 다음 측정 항목으로 구체화했다.

- 1일차 쉬움: 자동 전투로 승리, 왕좌 피해 없음, 큰 지시 없이 종료.
- 2일차 보통: 도둑이 위협으로 작동하되, 방 지시가 전투 시간과 도둑 저지에 유리해야 함.
- 3일차 어려움: 자동 전투는 패배하거나 매우 위험해야 하고, 방 지시와 임프 스킬을 쓰면 안정적으로 승리해야 함.

측정 지표:

- 전투 결과, 종료 시간, 왕좌 HP, 다운된 몬스터 수
- 처치한 적 수 / 등장한 적 수
- 도둑의 보물방 도달 여부와 훔치기 성공 여부
- 스킬 사용 횟수, 전투 후 자원량

## 구현 내용

- `scripts/units/Unit.gd`
  - `CharacterBody2D` 유닛 콜라이더를 반경 14px 원형으로 축소.
  - 몬스터와 적은 서로 물리 충돌하고, 같은 진영은 물리 충돌 대신 부드러운 회피만 적용.
  - 도둑 포함 적 유닛이 몬스터를 관통하지 못하도록 충돌 레이어/마스크 분리.
  - 충돌 시 옆 방향 우회 지점을 잠시 잡아 막힌 유닛을 돌아가도록 처리.
  - 다운된 유닛은 충돌체를 비활성화해 길막 잔상이 남지 않게 처리.

- `scripts/game/CombatSceneController.gd`
  - 적이 현재 방의 몬스터를 추격할 때 최종 목표 방을 덮어쓰지 않도록 수정.
  - 이 버그 때문에 1일차 탐험가가 입구 몬스터를 잡은 뒤 `goal_room=entrance`로 남아 전투가 끝나지 않는 문제가 있었다.
  - 가시 복도 `trap_lure` 지시 중 함정 피해/둔화 강화: 피해 24, 둔화 3.5초, 속도 계수 0.55.
  - 임프 스킬 개입 가치를 올리기 위해 화염구 피해 38, 화염 지대 피해 22 및 둔화 2.5초/0.7로 조정.

- `data/enemies.json`, `data/waves.json`
  - 3일차 자동 전투가 쉬운 문제를 보정하기 위해 수련생 용사 HP/ATK/DEF를 300/24/7로 조정.
  - 3일차 탐험가 수를 2명에서 3명으로 조정.
  - 수련생 용사 등장 시간을 8.0초에서 6.5초로 조정.

- `tools/BalanceSimulation.gd`, `tools/BalanceSimulation.tscn`
  - 밸런스 측정용 헤드리스 시뮬레이션 도구 추가.
  - 전체 시나리오와 `--scenario=DAY3_AUTO` 같은 단일 시나리오 실행을 지원.
  - 패배/타임아웃/도둑 훔치기 발생 시 유닛 스냅샷과 최근 로그를 JSON에 포함.

- `tools/DemoSmokeTest.gd`
  - 도둑이 작은 유닛 콜라이더를 관통하지 않고 우회하는 자동 검증 추가.

## 최종 측정 결과

명령:

```powershell
godot --headless --path . --scene res://tools/BalanceSimulation.tscn
```

결과:

| DAY2_AUTO | WIN | 27.1 | 1500 | 0 | 3/3 | N | N | 0 | 1443 | 380 |

해석:

- 1일차는 자동 승리와 왕좌 무피해를 만족한다. 다만 슬라임 1기가 다운되므로 실제 플레이에서 너무 거칠게 느껴지는지 확인 필요.
- 2일차는 자동도 승리하지만, 함정 유도 지시가 전투 시간을 27.1초에서 19.4초로 줄여 명확한 이득을 준다.
- 3일차는 자동 전투로 패배하고, 함정 유도와 임프 스킬 보조를 사용하면 승리한다. 체크리스트의 "직접 조종/임프 스킬 필요" 목표에 맞는다.

## 남은 판단 사항

- 1일차 자동 전투에서 슬라임이 쓰러지는 것이 "쉬움" 체감에 맞는지 실제 플레이로 확인.
- 스킬 피해량은 현재 코드 하드코딩이므로, 다음 단계에서 `data/skills.json`으로 빼면 밸런스 반복 속도가 좋아진다.
- 도둑 훔치기 게이지/남은 시간 UI가 아직 명확하지 않다. 2일차 플레이 감각을 보려면 UI 피드백 추가가 우선이다.
- 현재 시뮬레이션은 결정론적 단일 실행이다. 랜덤 요소를 넣게 되면 반복 횟수 기반 평균/최악값 측정으로 확장해야 한다.
