# 3차 업데이트 Phase 2 완료 기록

## 결론

저장 v3를 v4로 변환하는 경로와 안전 기록·복구·손상 격리 규칙을 구현했다. 진행 중인 v3 회차는 `update3_enabled=false`로 변환하므로 심장방이나 3차 전투 콘텐츠가 중간에 삽입되지 않는다.

## 구현 범위

- v4 envelope와 profile/active_run 신규 필드
- 정상 v3 5종 변환
- 기존 엔딩·계약·추억 자료 보존
- 새 게임과 새 회차의 profile/active_run 초기화 분리
- `.tmp`/`.bak` 중단 기록 복구
- 손상 v4 `.corrupt` 격리
- 반복 복원 멱등성(같은 입력을 여러 번 복원해도 결과가 같음)

## 검증 결과

- Phase 2 전용 저장 검사: 40/40 PASS
- 2차 저장 v3 회귀: 17/17 PASS
- 기존 캠페인 저장·불러오기: 233/233 PASS
- 2차 출시 계약 회귀: 383/383 PASS
- 핵심 빠른 검사: 15/15 PASS
