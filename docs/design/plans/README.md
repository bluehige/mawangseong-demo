## 후속 업데이트 원칙

1. 안정 버전의 정확한 상태는 이동하지 않는 SemVer 태그로 고정한다.
2. 다음 버전은 최신 `main`에서 `release/v0.N`을 만들고, 개별 Phase는 `codex/vNN-<topic>`에서 진행한다.
3. 원문 요구와 현재 저장소가 충돌하면 기준선 감사 문서에 차이와 선택 이유를 먼저 기록한다.
4. 신규 그래픽은 GPT 내부 이미지 생성만 사용하고, 생성 원본과 런타임 자산 및 `SOURCE.md`를 분리한다.
5. 자동 강제 배정 결과는 실제 플레이어 선택률로 표현하지 않는다. 자동 매트릭스와 외부 플레이 관측 자료를 별도 증거로 유지한다.
6. 버전 최종 후보에서만 사용자 요청 범위의 전체 스토리·그래픽·밸런스·버그 검수를 수행하고, 검수 SHA 이후에는 핸드오프 문서 외 제품 파일을 변경하지 않는다.
