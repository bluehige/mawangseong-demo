## 0. Codex 실행 계약
이 문서는 6차 업데이트의 아이디어 모음이 아니라 구현 범위, 데이터 계약, 작업 순서, 검수 기준을 고정하는 작업 계약이다. 아래 규칙은 구현 편의나 새 아이디어보다 우선한다.

1. 6차는 DAY 31 이후를 추가하지 않는다.
2. 6차는 새로운 30일 캠페인을 만들지 않는다.
3. 6차는 5차의 7전투 고정 일정 `alliance_week`를 복제하지 않는다.
4. 신규 모드는 `chronicle_expedition`이며, **8페이지 원정** 구조로 만든다.
5. 한 원정은 일반 전투 5회, 관리·사건 페이지 2회, 최종 보스전 1회로 제한한다.
6. 목표 플레이 시간은 첫 완료 50~70분, 반복 완료 40~60분이다.
7. 과거 회차는 적 유닛 복제나 이름 복제로 재현하지 않는다. 전술 태그, 사건, 배너, 보급 카드로만 표현한다.
8. 푸딩·곱·핀 등 이름 있는 플레이어 몬스터가 적으로 등장하는 일은 없다.
9. 회차 메아리는 완료 저장 전체를 복사하지 않고 6KB 이하의 요약 카드로 만든다.
10. 메아리 카드는 최대 24개, 고정 보관은 최대 6개다.
11. 메아리 압박은 한 원정에 최대 2개이며, 동일 카운터 영역을 중복 적용하지 않는다.
12. 메아리 압박 때문에 특정 몬스터가 없으면 자동 패배하는 구조를 만들지 않는다.
13. 신규 영구 플레이어블 몬스터는 정확히 2종만 추가한다: 딸깍, 바스락.
14. 로로는 원정 지휘관으로 강화하지만 전투 유닛으로 전환하지 않는다.
15. 신규 일반 적은 기록청 계열 6종만 추가한다.
16. 신규 최종 보스는 대기록관 아르카 1명만 추가한다.
17. 신규 분성은 정확히 3곳만 추가한다.
18. 세 분성은 모두 단층 고정 맵으로 제작한다. 3층, 무한 층, 자유 증축은 추가하지 않는다.
19. 원정 편성은 6종 선발, 전투당 4종 출전으로 고정한다.
20. 출전하지 않은 예비 몬스터는 불이익 대신 `준비됨` 보너스를 얻는다.
21. 전투 불능은 영구 사망, 유대 하락, 프로필 자원 손실을 만들지 않는다.
22. 선배–후배 계승은 스탯 상속이 아니라 행동형 사이드그레이드 기술 1개를 해금하는 구조다.
23. 교관 몬스터는 소모·은퇴·합성되지 않는다.
24. 한 몬스터는 계승 기술 1개만 장착한다.
25. 계승 기술에는 항상 효과 제한 또는 명확한 대가가 있다.
26. 모든 종족 조합을 자동 생성하지 않는다. 역할 태그와 허용 목록으로 호환성을 검증한다.
27. 몬스터 칭호는 전투 능력치를 제공하지 않는다.
28. 실키 왕관 진화 C13만 추가한다. 딸깍·바스락의 왕관 진화는 구현하지 않는다.
29. 전술 기록 재생은 완전한 결정론적 리플레이가 아니라 의미 이벤트 기반 요약 재생으로 제한한다.
30. 온라인 고스트, 온라인 랭킹, 시즌 종료, 서버 저장, PvP는 구현하지 않는다.
31. 신규 엔딩은 E29~E34의 6개만 추가한다.
32. 기존 E00~E28을 삭제하거나 하위 엔딩으로 취급하지 않는다.
33. 저장은 v7로 올린다. 진행 중 v6 회차에 6차 규칙을 강제로 삽입하지 않는다.
34. `GameRoot.gd`와 `CombatSceneController.gd`에 분성·메아리·교관·보스 ID별 거대 분기문을 추가하지 않는다.
35. 데이터 계약과 순수 서비스 테스트를 먼저 통과한 뒤 화면과 전투에 연결한다.
36. 신규 몬스터와 적은 placeholder 행동으로 역할 A/B 검증을 통과한 뒤 최종 그래픽을 제작한다.
37. 한 Phase에서 신규 몬스터 2종, 적 6종, 보스, 분성 3곳을 동시에 만들지 않는다.
38. 각 Phase 종료 시 변경 파일, 결정 로그, 자동 검사, 수동 캡처를 남긴다.
39. 1920×1080과 1366×768 UI 회귀 검사를 유지한다.
40. 기존 30일 캠페인, 마계 의회, 연맹 주간, 도전 공방의 저장과 완주를 모두 회귀 검증한다.

## 1. 선행 조건과 시작점
### 1.1 5차 업데이트 완료 인정 조건
- 저장 v6가 정상 저장, 임시 파일 복구, 백업 복원, 손상 격리를 모두 통과한다.
- `front_chronicle`, `council_season`, `alliance_week`가 서로의 프로필을 손상시키지 않고 완주된다.
- 연맹 주간의 평균 완료 시간이 40~85분 범위에 들어온다.
- 규격화 편성에서 기존 장기 레벨이 승률을 직접 결정하지 않는다.
- 게스트 성 3곳이 서로 다른 편성 판단을 만든다.
- 도전 공방의 검증기가 경로 단절, 목표 누락, 보스 중복, 적 과밀 조합을 차단한다.
- C01~C12 왕관 가운데 특정 1종 선택률이 60%를 넘으면 원인이 분석·수정됐다.
- L01~L12 합동기 가운데 특정 1개가 모든 조합에서 최적이 아니다.
- 생활 장면과 장식은 전투 능력치를 제공하지 않는다.
- E23~E28이 서로 다른 플레이 기록으로 도달 가능하다.
- 1366×768에서 주간 보드, 생활 화면, 도전 공방, 삼중 공성 HUD가 겹치지 않는다.
- 자동·수동·UI·저장·성능 검증이 모두 PASS다.

### 1.2 6차가 해결할 문제
1. 여러 회차를 완료해도 과거 플레이는 엔딩 도감과 수치 통계로만 남고 다음 전투에 서사적으로 돌아오지 않는다.
2. 플레이어가 오래 키운 선배 몬스터와 새로 쓰기 시작한 후배 몬스터 사이의 관계가 없다.
3. 로스터가 넓어졌지만 저사용 몬스터를 다시 꺼낼 직접적인 동기가 부족하다.
4. 연맹 주간은 짧고 편리하지만 일정이 고정돼 있어 탐험·발견 감각은 약하다.
5. 도전 공방은 조합 실험 공간이지만 세계관 안에서 왜 과거 전투를 다시 시험하는지 설명이 약하다.
6. 실키만 왕관 진화가 없어 13종 왕관 도감이 완성되지 않는다.
7. 로로의 원정 지휘관 정체성이 장기 업데이트에서 충분히 회수되지 않았다.
8. 전투 후 결과 화면은 풍부하지만 플레이어가 “그때 그 장면”을 다시 보는 기능이 없다.
9. 엔딩이 많아질수록 서로 다른 결말을 모두 정본 하나로 정리하려는 세계관 갈등이 필요하다.
10. 반복 보상을 더 이상 영구 공격력·HP로 늘릴 수 없으므로 관계·행동·기록 중심 보상이 필요하다.

## 2. 업데이트 정체성과 목표
### 2.1 업데이트 명칭
**6차 대형 업데이트: 잊힌 분성과 이어지는 이름**

핵심 문장:

> “지난 회차는 끝난 기록이 아니라, 다음 부하에게 길을 가르치는 오래된 발자국이다.”

### 2.2 플레이어 감정 목표
1. 과거의 엔딩과 빌드가 다음 원정의 사건과 적 전술로 되돌아오는 경험을 한다.
2. 오래 키운 몬스터가 다른 몬스터에게 행동을 가르치는 모습을 보며 관계가 세대처럼 이어진다고 느낀다.
3. 늘 쓰던 상위 편성 대신 평소 덜 쓰던 몬스터를 꺼내도 분명한 보상이 생긴다.
4. 로로가 단순 원정 메뉴 안내가 아니라 분성 탐험의 실제 지휘관으로 보인다.
5. 세 잊힌 분성이 서로 다른 경로·환경·스킬 판단을 요구한다.
6. 딸깍과 바스락이 기존 몬스터를 밀어내지 않고 새로운 전술 빈칸을 채운다.
7. 대기록관 아르카와 싸우며 “하나의 정답 빌드”보다 여러 결말이 공존하는 것이 게임의 핵심이라는 주제를 확인한다.
8. 전투 기록 재생에서 자신이 기억하지 못한 구조·마무리·합동 장면을 다시 본다.
9. E29~E34를 통해 분성 복구, 계승, 저사용 편성, 왕관 도감, 아르카 화해라는 서로 다른 목표를 계획한다.

## 4. 연대기 원정 전체 루프
### 4.1 해금 규칙

- 첫 진입 시 로로와 바티가 기록청 봉인 통지서를 읽는 5줄 장면을 재생한다.
- 첫 원정에서는 `종소리 기록고`만 선택 가능하다.
- 첫 완료 뒤 `침수된 필사실`, 두 번째 완료 뒤 `거울 뼈서고`를 해금한다.

### 4.2 시작 준비 6단계
| 단계 | 선택 | 제한 |
| --- | --- | --- |
| 1. 분성 선택 | 3곳 중 1곳 | 해금된 분성만 |
| 2. 원정대 선발 | 몬스터 6종 | 중복 instance 금지 |
| 3. 기본 전력 | 심장 1, 왕관 0~1, 합동기 0~1 | 합동기 구성원은 선발에 포함 |
| 4. 교관 지정 | 선배 1, 후배 1 | 호환 태그와 조건 충족 |
| 5. 메아리 선택 | 지원 메아리 1, 도전 메아리 1 | 같은 echo 중복 금지 |
| 6. 로로 노선 | 길잡이 대장 / 소문상 | 프로필에서 선택한 성장 경로 |

### 4.3 8페이지 일정
| 페이지 | 유형 | 핵심 내용 |
| ---: | --- | --- |
| 1 | 전투 | 분성 입구와 기본 규칙 학습 |
| 2 | 전투 분기 | 서로 다른 적 역할·보상을 가진 두 전투 경로 중 하나 선택 |
| 3 | 전투 | 도전 메아리 전술이 처음 완전 적용 |
| 4 | 관리 | 교관 수업, 예비 교체, 기록 법칙 선택 |
| 5 | 전투 | 분성 고유 보조 목표를 포함한 엘리트전 |
| 6 | 전투 | 기록청 봉쇄대와 분성 핵심 방어 |
| 7 | 사건 | 아르카의 정본 제안, 최종 편성 변경 |
| 8 | 보스 | 대기록관 아르카 3단계 전투 |

- 정확한 전투 수는 일반 전투 5회와 최종 보스전 1회다.
- 기본 완료 조건은 페이지 8 승리다.
- 페이지 2의 두 전투 경로는 threat 총량은 비슷하게 유지하고 적 역할·보조 목표·기록 조각 구성을 다르게 한다.
- 각 페이지는 전투 전 안전 지점과 전투 후 결산 안전 지점을 갖는다.

### 4.4 페이지 사이 루프
```text
페이지 결과
→ 몬스터 기여·교관 수업 진행 확인
→ 출전 4종 / 예비 2종 교체
→ 준비됨 토큰·긴장 표식 확인
→ 기록 조각으로 일회성 보급 선택
→ 다음 페이지와 메아리 압박 예고
→ 안전 저장
→ 다음 전투 또는 사건
```

### 4.5 목표 플레이 시간
| 구간 | 목표 시간 |
| --- | ---: |
| 준비 | 5~8분 |
| 일반 전투 1회 | 4~7분 |

### 4.6 실패와 재도전
- 정상 난이도는 현재 전투 직전 안전 저장에서 재도전한다.
- 재도전은 영구 자원·유대·메아리를 손실시키지 않는다.

- 선택형 `무수정 기록` 도전 인장을 켠 경우에만 원정 전체 재시작을 요구한다.

## 5. 원정 편성과 순환
### 5.1 편성 규칙
- 선발 몬스터: 6종
- 전투 출전: 4종
- 예비: 2종
- 왕관: 최대 1종
- 합동기: 최대 1개
- 계승 기술 장착: 몬스터당 최대 1개
- 같은 species 또는 instance 중복 금지
- 로로는 전투 편성 칸을 사용하지 않는다.

### 5.2 규격화
- 기준 레벨: 8
- 프로필에서 해금한 특화·진화·왕관·합동기·계승 기술은 사용 가능하다.
- 장기 캠페인의 금화·마력·식량·시설 단계는 반영하지 않는다.
- 분성별 고정 방과 임시 시설 2개만 사용한다.
- 스탯은 기준 레벨과 선택 성장만 계산한다.

### 5.3 `준비됨` 토큰
- 한 전투를 예비로 쉰 몬스터는 `prepared = true`를 얻는다.
- 다음 출전 시 다음 효과를 한 번 받는다.
  - 첫 액티브 스킬 재사용 대기 진행률 40%에서 시작
  - 처음 적과 교전할 때 최대 HP 10% 보호막, 4초
- 토큰은 최대 1개다.
- 전투 불능 이력이 있는 몬스터는 한 페이지 동안 준비됨을 얻지 못한다.
- 준비됨은 원정 종료 시 사라진다.

### 5.4 긴장 표식
- 전투 불능이 된 몬스터는 `strain = 1`을 얻는다.
- strain은 능력치를 직접 깎지 않는다.
- strain 상태에서는 다음 페이지의 교관 수업 주행동 담당이 될 수 없다.
- 한 페이지 예비로 쉬거나 페이지 4 관리에서 휴식시키면 제거된다.
- strain 최대치는 1이다.
- 영구 부상, 치료 재화, 현실 시간 대기는 없다.

### 5.5 편성 다양성 목표
- 동일 원정 내 6종 모두 최소 한 번 출전하는 비율 60% 이상을 목표로 한다.
- 프로필 사용률 하위 절반 몬스터가 원정 선발에 포함되는 비율 45% 이상을 목표로 한다.
- 특정 4종 고정 출전 조합이 정상 난이도 완료의 50%를 넘으면 원인을 분석한다.
- 저사용 몬스터를 강제 출전시키는 일일 임무는 만들지 않는다.

## 6. 회차 메아리 아카이브
### 6.1 목적
과거 회차를 단순 통계가 아니라 다음 원정의 전술적·서사적 재료로 사용한다. 메아리는 플레이어의 정확한 세이브 복사본이 아니라 “그 회차에서 어떤 성을 어떤 방식으로 지켰는가”를 요약한 카드다.

### 6.2 생성 시점
다음 모드의 승리 완료 시 메아리 카드를 생성한다.

- `front_chronicle`
- `council_season`
- `alliance_week`
- `chronicle_expedition`

패배·중도 포기 기록은 전술 압박 메아리로 만들지 않는다. 단, 플레이어가 원하면 패배 기록 한 개를 `교훈 카드`로 보존할 수 있으며 전투 효과는 없다.

### 6.3 저장 필드
```text
run_echo
├─ echo_id
├─ version
├─ source_mode
├─ source_ending_id
├─ source_castle_or_front_id
├─ dominant_archetype
├─ secondary_archetype
├─ top_monster_instance_ids[3]
├─ heart_id
├─ crown_id
├─ duo_link_id
├─ facility_focus_tags
├─ directive_distribution
├─ direct_control_ratio_band
├─ security_grade
├─ objective_loss_band
├─ clear_timestamp
├─ display_title
└─ pinned
```

다음 값은 저장하지 않는다.

- 전체 방 배치 원본
- 전체 전투 로그
- 몬스터 정확한 레벨·EXP
- 금화·마력·식량 잔액
- 픽셀 좌표
- 사용자 임의 텍스트
- 전체 대사 상태

### 6.4 크기와 보관 정책
- 카드당 UTF-8 JSON 6KB 이하
- 전체 최대 24개
- 플레이어 고정 보관 최대 6개
- 고정되지 않은 가장 오래된 카드부터 제거
- 동일 source mode·ending·archetype 조합이 3개를 넘으면 가장 대표성이 낮은 카드를 우선 정리
- 삭제 전 연대기 요약에는 총 완료 횟수만 남긴다.

### 6.5 원정에서의 사용
- 지원 메아리 1개: 일회성 기억 보급 카드 하나를 제공한다.
- 도전 메아리 1개: 압박 태그 1~2개와 사건 문구를 제공한다.
- 지원·도전 메아리는 같은 카드일 수 없다.
- 플레이어는 압박 태그와 예상 적 역할을 원정 시작 전에 본다.
- 로로의 길잡이 대장 경로는 도전 메아리를 한 번 교체할 수 있다.
- 소문상 경로는 더 어려운 태그를 유지하는 대신 기록 조각 보상을 늘릴 수 있다.

### 6.6 기본 메아리
과거 완료 기록이 없는 프로필에는 바티가 만든 가상 훈련 기록 6개를 제공한다.

1. 느린 방벽형
2. 빠른 총공격형
3. 회복 교대형
4. 보물 추격형
5. 함정 제어형
6. 균형 혼성형

기본 카드는 플레이어의 과거로 표시하지 않고 `바티의 모의 기록`으로 명시한다.

### 6.7 이름 있는 몬스터 보호
- 메아리 전투에서는 과거 몬스터의 초상·이름·스프라이트를 적으로 사용하지 않는다.
- `푸딩의 메아리` 같은 적 유닛을 만들지 않는다.
- 메아리는 깃발, 방 배치 힌트, 적 병종 구성, 기록청 보고서로만 표현한다.
- 과거 top monster는 “전열 중심 기록”, “추격 중심 기록” 같은 역할 문구로 변환한다.

## 8. 잊힌 분성 3곳
### 8.1 공통 규칙

- 방 수는 8~10개다.

- 보조 목표가 파괴돼도 즉시 원정 실패가 되지 않는다.
- 임시 왕좌 HP 0만 해당 전투 패배다.

### 8.2 BC01 종소리 기록고
#### 전장 정체성
기록청이 모든 문을 종소리 순서에 따라 열고 닫는 분성이다. 플레이어는 종 주기를 읽어 방어 병력을 미리 옮기거나, 문이 닫히기 전에 후퇴선을 정해야 한다.

#### 맵 구조
- 방 9개
- 문 그룹 A/B/C 각 2개
- 중앙 종핵 1개
- 임시 왕좌 1개
- 우회 통로 2개

#### 고유 규칙: `bell_cycle`
- 18초 주기로 A→B→C 문 그룹이 활성화된다.
- 닫히는 문턱에 유닛이 있으면 0.8초 유예 뒤 닫힌다.
- 필수 경로가 완전히 끊기는 경우 문은 파괴 가능한 기록문으로 전환된다.
- 적과 아군 모두 다음 주기를 5초 전에 HUD에서 확인한다.
- 딸깍은 주기를 무시하지 않고, 한 문 그룹의 닫힘을 최대 4초 늦추거나 앞당긴다.

#### 보조 목표
- 종핵 HP 50% 이상 유지
- 문 파괴 2회 이하
- 같은 방에 몬스터 4종을 10초 이상 몰아넣지 않기

#### 유효 편성
- 딸깍, 실키, 곱, 포포, 푸딩

#### 비필수 원칙
딸깍이 없어도 종 주기 예고와 기존 방 지침으로 완료할 수 있어야 한다.

### 8.3 BC02 침수된 필사실
#### 전장 정체성
마력 잉크 수위가 오르내리며 저지대 방의 이동과 화염 지속시간을 바꾼다. 실제 유체 시뮬레이션은 사용하지 않고 방 상태를 전환한다.

#### 맵 구조
- 방 10개
- 저지대 4개
- 고가 필사대 3개
- 배수 레버 방 1개
- 기록 핵 1개
- 임시 왕좌 1개

#### 고유 규칙: `ink_tide`
- 20초마다 `low`, `rising`, `high`, `falling` 네 상태 순환
- high 상태 저지대: 이동 속도 -20%, 화염 지대 지속시간 -20%
- high 상태 고가 필사대: 원거리 사거리 +8%
- 배수 레버를 직접 조종 또는 방 지침으로 작동하면 다음 high 구간을 5초 단축
- 적도 저지대 둔화를 받는다.

#### 보조 목표
- 기록 묶음 3개 중 2개 보존
- 배수 레버 2회 이상 작동
- 임프 화염 지대만으로 모든 전투를 해결하지 않기

#### 유효 편성
- 바스락, 모리, 베베, 포포, 핀

#### 비필수 원칙
화염 계열이 약화되지만 완전 면역은 없다. 핀을 편성해도 역할을 잃지 않는다.

### 8.4 BC03 거울 뼈서고
#### 전장 정체성
세 쌍의 거울 방이 연결돼 첫 스킬 효과를 반대쪽 방에 약하게 복제한다. 아군과 적 모두 이용할 수 있어 스킬 사용 순서가 중요하다.

#### 맵 구조
- 방 8개
- 거울 연결 방 3쌍
- 비연결 왕좌 방 1개
- 기억 성물 1개

#### 고유 규칙: `mirror_reference`
- 거울 쌍마다 12초마다 한 번 첫 액티브 스킬을 기록한다.
- 1.5초 뒤 반대쪽 방에 40% 위력으로 복제한다.
- 피해·회복·보호막·둔화만 복제한다.
- 직접 이동, 처치, 소환, 영구 잠금은 복제하지 않는다.
- 적의 첫 특수 기술도 같은 규칙을 사용한다.
- HUD에 어느 방 쌍이 준비됐는지 표시한다.

#### 보조 목표
- 적 스킬 거울 복제 3회 이하
- 아군 유효 복제 4회 이상
- 기억 성물 HP 40% 이상

#### 유효 편성
- 바스락, 루미, 두둠, 모리, 핀

#### 비필수 원칙
거울 복제는 강한 스킬을 두 배로 만드는 구조가 아니다. 모든 수치는 40% 상한과 중첩 제한을 적용한다.

### 8.5 분성 숙련
각 분성은 다음 세 도장을 갖는다.

1. 첫 복구
2. 보조 목표 2개 이상 완료
3. 서로 다른 8종 이상으로 완료

숙련 보상은 장식, 칭호, 도전 공방 맵 해금이다. 전투 스탯은 제공하지 않는다.

## 9. 선배–후배 교관 계승
### 9.1 목적
오래 키운 몬스터가 단순히 높은 레벨의 전투 유닛으로 끝나지 않고, 다른 몬스터의 행동에 흔적을 남기게 한다. 계승은 숫자 상속이 아니라 전투 판단 하나를 배우는 과정이다.

### 9.2 교관 조건
- 유대 80 이상
- 승리 완료 기록 2회 이상
- 현재 원정 선발 6종에 포함
- 해당 원정에서 strain이 없는 상태로 시작
- 가르칠 수 있는 역할 태그 보유

### 9.3 후배 조건
- 유대 30 이상
- 해당 계승 기술 미해금
- 교관과 동일 instance가 아님
- 허용된 수강 역할 태그 보유
- 현재 원정 선발 6종에 포함

### 9.4 수업 진행
한 원정에서 정확히 세 단계다.

1. **관찰**: 교관과 후배가 같은 전투에 출전해 교관 행동 3회 확인
2. **모방**: 후배가 지정 행동을 4회 성공
3. **독립**: 페이지 5 이후 전투에서 교관 도움 없이 지정 행동 2회 성공

- 각 단계는 UI에 정확한 행동 문구로 표시한다.
- “좋은 플레이를 하세요” 같은 추상 목표는 사용하지 않는다.
- 보스전 전에 2단계까지만 완료돼도 페이지 8에서 마지막 기회를 제공한다.
- 실패한 원정은 최대 2단계의 `수업 노트`를 보존한다.
- 같은 교관–후배–기술 조합의 수업 노트만 이어진다.

### 9.5 해금과 장착
- 수업 완료 시 후배 instance에 계승 기술을 영구 해금한다.
- 해금과 장착은 분리한다.
- 몬스터당 장착 1개다.
- 원정 준비, 캠페인 새 회차 준비, 연맹 주간 준비에서만 변경한다.
- 진행 중 전투에서는 변경하지 않는다.
- 계승 기술을 장착하지 않아도 불이익이 없다.

### 9.6 호환성
- 기술은 `teacher_tags`와 `student_tags`로 호환성을 결정한다.
- 모든 pair를 데이터로 열거하지 않는다.
- 단, 시스템적으로 문제가 있는 조합은 `blocked_pairs`에 명시한다.
- 같은 기술을 여러 교관에게 배울 수 있지만 해금 결과는 하나다.
- 교관별 대사와 관계 카드는 다르게 기록한다.

### 9.7 파워 예산
- 계승 기술 하나가 특정 유효 시나리오에서 결과를 5~12% 바꾸는 수준을 목표로 한다.
- 기본 공격력·HP·방어의 영구 고정 상승은 없다.
- 한 기술이 여러 시스템을 동시에 강화하지 않는다.
- 왕관·합동기·심장·계승 기술의 총 피해 증폭은 기존 전투 상한을 넘지 않는다.

#### `canonical_index`

### 17.4 2단계: 삭제 도장

### 17.5 3단계: 모순 허용

## 20. 전술 기록 재생
### 20.1 목표
전투를 완전히 다시 계산하지 않고, 플레이어가 기억할 만한 장면을 방 그래프 위에서 짧게 재생한다.

### 20.2 기록 대상 이벤트
- 몬스터 방 이동
- 왕좌·보조 목표 피해
- 전투 불능·구조
- 액티브·왕관·합동기·심장 사용
- 문 잠금·해제
- 시설 무력화·복구
- 적 특수 기술
- 교관 수업 행동
- 보스 단계 전환
- 마지막 처치

### 20.3 저장 형식
```text
tactical_event
├─ t
├─ type
├─ actor_id
├─ target_id
├─ room_id
├─ target_room_id
├─ value_band
├─ skill_or_rule_id
└─ importance
```

- 픽셀 좌표는 저장하지 않는다.
- value는 정확한 전체 피해 로그 대신 작은 구간값으로 저장 가능하다.
- 전투당 최대 400개 이벤트
- 최종 보스 최대 600개
- 프로필에는 최근 원정 재생 3개만 보관
- 재생 데이터 1개 최대 200KB

### 20.4 재생 화면
- 분성 방 그래프 단순화 표시
- 몬스터·적 초상 아이콘 이동
- 시간축 0.5× / 1× / 2×
- 중요 이벤트 자동 멈춤 옵션
- 교관 수업 순간 북마크
- 마지막 30초 바로 보기
- 실제 전투 결과와 다른 계산은 하지 않는다.

### 20.5 명칭 주의
UI에는 `전술 기록 재생`으로 표시한다. `완전 리플레이`, `영상 녹화`, `정확한 재현`이라고 부르지 않는다.

### 20.6 실패 방지
- 기록 서비스 오류가 전투를 중단시키지 않는다.
- 저장 크기 초과 시 중요도 낮은 기본 공격 이벤트부터 제거한다.
- 재생 데이터 손상은 해당 재생만 격리한다.
- active run 복원에는 재생 데이터가 필수가 아니다.
