## 1. 목표

이번 수정은 전투를 개별 유닛 조작 게임에서 지시 중심의 자동 전투로 바꾸고, 전투의 답답함과 초기 조작 부담을 함께 줄이면서 확대된 텍스트로 깨진 UI 배열을 정상화하는 작업이다.

- 단일 유닛 직접 이동·공격·스킬 조작을 제거하고 지시만으로 전투를 운영한다.
- 유닛끼리 밀고 막는 충돌과 회피 우회를 제거한다.
- 전투 속도를 최대 3배까지 선택할 수 있게 한다.
- PC와 모바일의 초기 조작법 및 튜토리얼을 짧고 직관적으로 다시 구성한다.
- 확대된 폰트 크기는 가독성 기준으로 유지하면서 겹침, 잘림, 영역 이탈, 버튼 배열 붕괴를 수정한다.
- Windows PC에서 마왕 이름을 한글 IME로 입력하지 못하는 문제를 재현하고 수정한다.

단순히 버튼을 숨기는 데 그치지 않는다. 직접 조작 제거 후에도 스킬 사용, 특수 유닛 행동, 튜토리얼, 테스트가 모두 지시 전용 규칙으로 일관되게 동작해야 완료로 본다.

## 2. 확정할 최종 조작 규칙

### 전투 중 허용되는 조작

- 전역 지시 선택: `사수`, `총공격`, `생존`
- 방 단위 지시 선택: `집중 방어`, `함정 유도`, `후퇴 지점`
- 전투 속도 선택: `x1`, `x1.5`, `x2`, `x3`
- 일시정지와 정보 확인
- 유닛 선택은 상태와 능력 확인 용도로만 유지

### 제거되는 조작

- 선택 유닛의 개별 이동 지점 지정
- 선택 유닛의 개별 공격 대상 지정
- 직접 조작 진입과 AI 복귀
- 개별 스킬 버튼과 숫자키 수동 발동
- 직접 이동·직접 공격을 요구하는 튜토리얼 단계

### 지시별 AI 계약

| 지시 | 이동·표적 원칙 | 스킬 우선순위 |
|---|---|---|
| 사수 | 배치된 방과 방어선을 유지하고 가까운 침입자 대응 | 보호막, 회복, 저지, 방어형 스킬 우선 |
| 총공격 | 전투 가능한 적을 적극 추격하고 고위협 대상을 우선 제거 | 공격, 광역, 마무리 스킬 우선 |
| 생존 | 위험 유닛은 후퇴 지점 또는 안전 지역으로 이동하고 구조 대상을 우선 | 회복, 구조, 생존기 우선 |

`입구 봉쇄`라는 표현은 충돌 제거 후 실제로 길을 막는다는 오해를 만들 수 있으므로 `집중 방어`로 변경한다. 공격 판정, 스킬 범위, 투사체 명중 판정은 유지하며 유닛 몸체 간 이동 충돌만 제거한다.

## 3. 현재 구조에서 확인된 영향 범위

- `scripts/units/Unit.gd`에 직접 조작 상태, 명령 지점·대상, 유닛 충돌 반경, 회피와 우회 이동이 함께 들어 있다.
- `scripts/game/GameRoot.gd`에 우클릭·터치 직접 이동/공격, 직접 조작 진입/해제, 수동 스킬 입력과 이름 입력 화면이 연결돼 있다.
- `scripts/game/CombatSceneController.gd`는 직접 조작 유닛의 AI를 우회하고, 현재 `combat_speed`를 시뮬레이션 델타에 적용한다.
- `scripts/ui/HUDController.gd`에는 PC·모바일 직접 조작, AI 복귀, 개별 스킬 버튼과 현재 속도 UI가 있다.
- 온보딩 데이터와 `TutorialManager`는 직접 조작 1회, 직접 공격 1회, 전투 로그 확인을 필수 단계로 사용한다.
- 일부 특수 유닛, 카메라, 영상 캡처와 스모크 테스트가 직접 조작 상태를 전제로 한다.
- 이름 입력은 `LineEdit`에 한글 폰트를 사용하고 저장 데이터도 유니코드를 보존한다. 다만 안내창을 닫을 때 화면과 입력창을 다시 만든 뒤 포커스를 잡고, 전역 키 입력도 동시에 처리하므로 Windows IME 조합 컨텍스트가 끊기는지 실제 네이티브 빌드에서 확인해야 한다.
- 공통 `UISettings`의 폰트 확대와 HUD의 자동 축소 보정이 적용돼 있지만, 여러 화면에 고정 `Rect2`, 고정 `custom_minimum_size`, `clip_contents`가 혼재한다. 이 구조에서는 긴 한글 문구나 작은 해상도에서 텍스트와 버튼이 겹치거나 잘릴 수 있으므로 전투 HUD만이 아닌 전체 주요 화면을 점검해야 한다.

## 5. 페이즈별 상세 계획

### Phase 0. 기준선 고정과 측정

1. 실제 구현은 현재 STOVE 준비 브랜치가 아니라 최신 `main`에서 새 `codex/vNN-directive-combat` 브랜치를 만든다.
2. 직접 조작, 충돌, 속도, 온보딩, 이름 입력과 폰트·레이아웃에 연결된 코드·데이터·테스트를 목록화한다.
3. 대표 전투 5개를 x1 기준으로 자동 측정하고 승패, 전투 시간, 사망 유닛, 남은 HP·마나를 기록한다.
4. 현재 측정 참고값은 DAY 1 41.7초, DAY 2 37.3초, DAY 3 57.4초, DAY 15 59.8초, DAY 30 71.3초다. 구현 브랜치에서 같은 조건으로 다시 측정해 공식 기준선으로 고정한다.

### Phase 1. Windows PC 한글 이름 입력 수정

#### 재현 절차

Windows 64-bit 네이티브 빌드와 Microsoft 한국어 IME를 사용한다. 에디터의 문자열 직접 대입만으로는 OS 조합 입력을 검증할 수 없으므로 수동 재현을 필수로 둔다.

- 안내창을 닫은 직후 `마왕` 입력 및 Enter 확정
- `푸른 달`, `검은성123`처럼 공백·숫자가 섞인 이름 입력
- 조합 중 Backspace, 방향키, 한/영 전환 후 재입력
- 버튼 클릭 확정과 Enter 확정을 각각 확인
- 저장 후 재실행하여 이름이 동일하게 복원되는지 확인

#### 수정 순서

1. 안내창을 닫을 때 이름 입력 화면 전체를 재생성하지 않고, 이미 생성한 `LineEdit`를 표시·활성화한 뒤 한 번만 포커스한다.
2. 텍스트 입력 컨트롤이 포커스를 가진 동안 전역 단축키와 게임용 `_input` 경로가 키 이벤트를 가로채지 않도록 입력 소유권을 분리한다.
3. IME 조합 중인 미확정 문자열은 변경하거나 검증하지 않고, 버튼 클릭 또는 `text_submitted` 시점의 확정 문자열만 검사한다.
4. 길이 제한은 조합 중간이 아니라 확정된 유니코드 문자열 기준으로 적용하며, 기존 12자 정책은 유지한다.
5. 이름 저장·불러오기에서 한글과 혼합 문자열이 보존되는 자동 회귀 테스트를 추가한다.

#### 완료 기준

- Windows 네이티브에서 초성·중성·종성이 정상 조합되고 글자가 분리되거나 사라지지 않는다.
- 마우스 확정, Enter 확정, Backspace, 한/영 전환이 모두 정상이다.
- PC Web Chrome은 보조 회귀 대상으로 확인한다.
- 헤드리스 테스트는 유니코드 검증·저장만 담당하고, 네이티브 IME 수동 확인을 대체하지 않는다.

### Phase 2. 단일 유닛 직접 조작 제거

1. `Unit.gd`의 직접 조작 상태와 이동·공격 명령 API를 제거한다.
2. `GameRoot.gd`의 우클릭·터치 직접 명령, 직접 조작 진입·해제, 수동 스킬 키 입력을 제거한다.
3. `CombatSceneController.gd`의 직접 조작 AI 우회와 직접 표적 우선 규칙을 제거한다.
4. 유닛 선택은 정보 표시만 수행하도록 입력 계약을 바꾼다.
5. 검색과 테스트로 `direct_control`, `command_move`, `command_attack`, 직접 스킬 호출의 남은 런타임 경로가 없는지 확인한다.

### Phase 3. 자동 스킬과 특수 유닛 이관

1. 모든 전투 스킬에 사거리, 최소 효율, 대상 우선순위, 마나 보존량, 재사용 대기 조건을 정의한다.
2. `사수`, `총공격`, `생존` 지시에 따라 같은 스킬도 사용 우선순위가 달라지게 한다.
3. 광역기는 적 밀집도, 회복기는 HP 비율과 구조 가치, 마무리기는 처치 가능성을 기준으로 발동한다.
4. Bebe, Toktok, 다층 카메라 등 직접 조작 예외를 지시형 AI와 자동 카메라 규칙으로 옮긴다.
5. 무한 추격, 마나 즉시 소진, 스킬 미사용 상태가 없는지 전투 로그와 테스트로 확인한다.

### Phase 4. 충돌 판정 제거

1. 아군·적 유닛 몸체의 물리 레이어, 마스크, 이동용 충돌 도형을 제거한다.
2. 유닛 간 회피 벡터, 우회 목적지, 우회 타이머와 충돌 반경 상수를 제거한다.
3. 내비게이션 경로, 방 경계, 던전 바깥 이탈 방지와 공격 사거리 판정은 유지한다.
4. 유닛이 한 점에 과도하게 포개져 식별하기 어려우면 충돌을 되살리지 않고 렌더 오프셋, 선택 윤곽, 체력바 배열로 가독성을 보완한다.
5. Toktok 돌진처럼 스킬 자체의 명중 판정은 이동 충돌과 분리해 유지한다.

### Phase 5. 최대 3배속

1. 전투 속도 선택지를 `x1`, `x1.5`, `x2`, `x3`으로 통일한다.
2. DAY 1~3 튜토리얼 중에는 x1만 허용하고 나머지 속도 버튼은 잠금 상태와 해금 조건을 표시한다.
3. 튜토리얼 완료 뒤 첫 일반 전투를 일시정지하고 속도 위치·배율·자동 판정 동기화를 설명하는 안내를 한 번만 보여 준다.
4. 안내 확인 여부를 저장하되 기존 저장에는 선택 필드로 취급해 불러오기 호환성을 유지한다.
5. `combat_speed`가 AI, 이동, 쿨다운, 상태 효과, 공격 지연, 투사체, 전투 타이머에 동일하게 적용되는지 정리한다.
6. 고정 실시간 Tween으로 구현된 근접 접촉 지연과 투사체 이동을 시뮬레이션 시간에 맞추거나 판정과 연출을 분리한다.
7. VFX와 SFX는 x3에서 겹침·누락·과도한 반복이 없는 수준으로 재생 정책을 조정한다.
8. 일시정지 후 복귀, 전투 종료 직전 속도 변경, 씬 전환에서 속도 상태가 누수되지 않게 한다.

선형 환산상 현재 대표 전투는 x3에서 약 12.4~23.8초 범위가 예상된다. 충돌 제거로 더 빨라질 수 있으므로 피해량을 먼저 높이지 않고 실제 측정 후 밸런스를 조정한다.

### Phase 6. 폰트 확대 UI 배열 정상화

폰트 크기를 이전 값으로 일괄 축소하는 방식은 사용하지 않는다. 읽기 쉬운 현재 기준을 유지하고, 화면이 텍스트 크기를 수용하도록 레이아웃을 고친다.

#### 점검 화면

- 타이틀, 마왕 이름 입력, 온보딩과 튜토리얼 안내
- 관리 화면의 자원·활동·시설·목록·결과 패널
- 일반 전투, 다층 전투, 특수 전투 HUD와 유닛 정보·전투 결과·팝업
- 지역·전선·심장·캠페인 선택 화면과 전초기지 관리·전투
- 연대기, 의회·왕관·결정 오버레이, 설정·확인 대화상자

#### 수정 순서

1. 각 화면을 `텍스트 잘림`, `버튼 겹침`, `패널 이탈`, `줄바꿈 실패`, `스크롤 불가`, `터치 영역 불일치`로 분류해 결함 목록을 만든다.
2. 공통 폰트 배율, 제목·본문·보조문·버튼의 최소 크기와 줄 간격을 역할별 토큰으로 정리한다.
3. 고정 좌표와 고정 높이를 필요한 곳만 남기고 `Container`, 앵커, 크기 플래그, 자동 줄바꿈, 콘텐츠 기반 최소 크기와 스크롤 영역으로 전환한다.
4. 자동 폰트 축소는 마지막 안전장치로만 사용하고, 본문과 핵심 버튼이 가독성 하한 아래로 작아지지 않게 한다.
5. 가장 긴 실제 한글 문구, 수치 자릿수가 큰 상태, 버튼이 가장 많이 나타나는 상태로 화면을 채워 확인한다.
6. 팝업과 튜토리얼 강조막이 수정된 컨트롤 위치 및 클릭·터치 영역을 정확히 따라가는지 확인한다.

#### 해상도 검수 기준

| 플랫폼 | 필수 확인 해상도·환경 | 확인 항목 |
|---|---|---|
| PC | 1920×1080, 1366×768, 1280×720 | 겹침·잘림·화면 이탈, 버튼 접근성, 스크롤 |
| PC Web | Chrome 1920×1080, 1366×768 | 브라우저 배율 100%, 캔버스 크기 변화 |
| 모바일 | 현재 대표 가로 해상도와 안전 영역이 있는 소형 화면 | 터치 목표, 긴 문구 줄바꿈, 하단 바·팝업 안전 영역 |

완료 기준은 모든 화면을 똑같은 크기로 만드는 것이 아니라, 정보 우선순위를 유지하면서 핵심 텍스트와 조작 요소가 겹치거나 잘리지 않는 것이다. 부가 설명이 긴 화면은 억지로 압축하지 않고 스크롤 또는 단계적 펼침을 사용한다.

### Phase 7. 조작 UI 단순화

PC 하단 전투 바의 기본 구조는 다음으로 제한한다.

`[사수] [총공격] [생존] | [상황별 방 지시] | [x1/x1.5/x2/x3] [일시정지]`

- 직접 조작, AI 복귀, 수동 스킬 영역을 삭제한다.
- 선택 유닛 패널은 HP, 역할, 현재 지시, 자동 스킬 상태만 간결하게 표시한다.
- 모바일도 같은 개념과 순서를 사용하되, 가로폭이 부족하면 속도 버튼 하나로 x1→x1.5→x2→x3을 순환한다.
- 선택된 지시와 적용 범위를 색상, 아이콘, 짧은 문구로 즉시 보여준다.
- 클릭·터치 목표 크기와 1366×768, 1920×1080, 대표 모바일 세로·가로 해상도를 확인한다.

### Phase 8. DAY 1~3 튜토리얼 재작성

| 날짜 | 필수 행동 | 안내 원칙 |
|---|---|---|
| DAY 1 | 슬라임 배치 → `사수` → 전투 시작 | 전투 중 필수 입력 없이 지시에 따라 AI가 싸운다는 점만 보여줌 |
| DAY 2 | 가시 복도 선택 → `함정 유도` → 전투 시작 | 방 지시가 전장 운영을 바꾼다는 점을 체험 |
| DAY 3 | 상황에 맞는 전역 지시 선택 → 전투 시작 | 스킬이 지시에 맞춰 자동 사용된다는 점을 보여줌 |

- 직접 조작 1회, 직접 공격 1회, 수동 화염구 사용을 삭제한다.
- 전투 로그 열기는 필수 게이트에서 빼고 선택 도움말로 둔다.
- 한 화면에는 한 행동만 강조하고, 현재 클릭 가능한 대상 외에는 강조를 최소화한다.
- 스킵·재시작·저장 복귀 시 튜토리얼 상태가 막히지 않는지 확인한다.

### Phase 9. 전투 밸런스와 관련 검증

#### 밸런스 원칙

- x3는 시뮬레이션 시간만 단축하고 동일한 지시와 난수 조건에서 x1과 승패가 달라지지 않아야 한다.
- 충돌 제거로 집중 화력이 과도해지면 공격력을 일괄 하향하지 않고 타깃 분산, 동시 공격 상한, AI 우선순위를 먼저 조정한다.
- 자동 스킬 때문에 마나가 초반에 소진되면 지시별 마나 보존량과 최소 효율 조건을 조정한다.
- 초반 DAY 1~3은 지시 선택 결과가 눈에 보여야 하지만 오답 한 번으로 즉시 패배하지 않게 한다.

- 지시 전환과 지시별 AI 행동 단위 테스트
- 자동 스킬 조건·마나 예산 테스트
- 유닛 간 이동 충돌이 없고 경계는 유지되는 테스트

## 8. 최종 완료 조건

- 플레이어가 전투 중 개별 유닛 이동·공격·스킬 명령을 내릴 수 없다.
- 전투의 모든 유닛 행동과 스킬이 전역·방 지시 및 AI 조건으로 결정된다.
- 유닛 몸체가 서로 막지 않으며 공격·스킬 명중과 맵 경계는 정상이다.
- x1~x3에서 시뮬레이션 결과가 동등하고 x3의 실제 전투 시간이 대체로 x1의 1/3 수준이다.
- DAY 1~3이 지시와 전투 시작만으로 진행되고 불필요한 필수 조작이 없다.
- Windows 네이티브에서 마왕 이름의 한글 조합, 수정, 확정, 저장·불러오기가 정상이다.
- 확대된 폰트의 가독성 기준을 유지하면서 주요 화면에 텍스트·버튼 겹침, 잘림, 화면 이탈과 접근 불가능한 조작이 없다.
- 변경 범위 관련 테스트와 PC·Web·모바일 지정 해상도 UI 확인이 통과하고, 남은 위험이 핸드오프에 기록돼 있다.
