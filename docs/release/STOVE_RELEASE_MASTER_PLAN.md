# STOVE 판매·자체등급분류 출시 마스터 플랜

작성일: 2026-07-15
대상: `마왕님, 마왕성은 누가 지켜요?` Windows 유료 정식판
현재 판정: **계정·법적 주체 등록 대기 / 제출 자료 기반 준비 완료 단계**

## 이번 입점의 목적

1. STOVE에 Windows 정식판을 실제로 출시하고 판매한다.
2. STOVE의 자체등급분류를 이용해 대한민국 이용등급을 확보한다.
3. 등급 확정 뒤 게임물관리위원회에 `자체등급분류 게임물 유통 통보서`를 제출해
   같은 게임을 Steam에서도 대한민국에 유통할 수 있도록 준비한다.

STOVE에서 심의만 받고 게임을 출시하지 않는 방식은 사용하지 않는다. STOVE 자체
심의 체크리스트는 STOVE를 통한 실제 출시를 전제로 한다. 또한 STOVE에서 받은
등급은 Steam에 자동 반영되지 않으므로 별도 유통 통보가 필요하다.

## 공식 절차와 책임 분담

| 단계 | 사용자가 직접 할 일 | 저장소·자료 작업 |
|---|---|---|
| 1. 가입 | 영구 사용 이메일로 STOVE 계정 로그인, 약관 확인·동의 | 입력할 이름·ID 후보와 주의사항 제공 |
| 2. 구조 생성 | 그룹명·ID 최종 승인, Studio에서 그룹/프로젝트/상품 생성 | `stove/store/STUDIO_PORTAL_VALUES.md` 작성 |
| 3. 법적 정보 | 개인/사업자 선택, 신원·사업자·은행 서류 제출 | 민감정보를 저장하지 않고 공개 메타데이터만 관리 |
| 4. 상품 페이지 | 공개 이름·연락처·가격·출시일 최종 승인 | 문구·이미지·사양·태그·언어 표 작성 |
| 5. 빌드 | 포털 접근과 최종 업로드 승인 | 사운드 확정 뒤 Windows 빌드·실행 설정·업로드 준비 |
| 6. 자체심의 | 설문 사실 확인과 법적 제출 | 게임 설명서·언어 파일·CG 목록·영상·답변 초안 준비 |
| 7. 검수 | STOVE 피드백 확인·승인 | 반려 항목 수정, 다시 패키징·검증 |
| 8. 출시 | 가격·시간·자동/수동 출시 최종 확인 | 출시 직전 검증 결과 제공 |
| 9. Steam 통보 | 서명·제출 및 등급번호 확인 | 유통 통보서 입력값과 첨부자료 정리 |

비밀번호, 인증번호, 주민등록번호, 신분증, 통장 사본은 저장소나 채팅에 올리지
않는다. 약관 동의, 본인 확인, 은행 정보 제출과 최종 출시 승인은 사용자가 직접
수행한다.

## 1. STOVE Studio 가입과 구조 생성

- 가입: <https://studio.onstove.com>
- 대표자가 계속 관리할 이메일 계정을 사용한다.
- Studio 이용약관 동의가 입점 계약의 일부이므로 동의 전에 최신 약관과 유료
  서비스 계약을 직접 확인한다.
- 관리 구조는 `그룹 > 프로젝트 > 상품`이다. 계약과 정산은 프로젝트 단위로
  연결되며, 본편은 `BASIC` 상품으로 만든다.
- 프로젝트 ID와 게임/상품 ID는 생성 후 수정이 어렵거나 불가능하므로 임의로
  확정하지 않는다. 저장소의 추천안을 확인한 뒤 생성한다.

## 2. 개인/사업자와 은행 정보

자체등급분류와 정산을 받으려면 이 단계가 먼저 승인돼야 한다. 공식 가이드의 현재
준비 서류는 다음과 같고, 검토에는 최대 약 3영업일이 안내돼 있다.

- 개인: 3개월 이내 주민등록본 또는 여권, 통장 사본
- 개인/법인 사업자: 사업자등록증, 통장 사본
- 계좌번호·예금주와 통장 사본이 일치해야 한다.

개인으로도 등록할 수 있지만 유료 판매자의 법정 표시정보와 개인정보 노출 범위를
최신 등록 화면에서 반드시 확인한다. 주소·연락처 공개가 부담되거나 계속 게임을
판매할 계획이면 개인사업자 등록과 게임제작업/배급업·통신판매업 해당 여부를
세무사와 관할 기관에 먼저 확인하는 편이 안전하다. Steam을 개인 명의로 등록한
사실이 STOVE도 반드시 개인이어야 한다는 뜻은 아니다.

## 3. 상품 페이지

현재 공식 가이드 기준으로 준비할 핵심 항목은 다음과 같다.

- 노출 국가/지역과 지원 언어
- 한 줄 소개, 상세 설명, 최대 10개 태그
- 표시 이미지/영상 최대 20개
- 스크린샷: 860×483, JPG/PNG/GIF/WEBP
- 타이틀 이미지: 500×500 및 757×426
- Windows 바탕화면 아이콘: 256×256 ICO
- PC 썸네일: 500×500
- 최소 사양: 필수이며 영문으로 작성
- 고객지원/문의 링크: 필수. 다른 판매 플랫폼 링크는 넣지 않는다.

STOVE 문구와 업로드용 그래픽은 각각 `stove/store/`와 `marketing/stove/`에
준비한다. 시스템 요구사항은 실제 저사양 측정 전까지 초안으로만 둔다.

## 4. 빌드와 STOVE 연동 결정

현재 게임은 싱글 플레이 기본 상품이며 업적, 인앱 결제, DLC를 출시 시점에
약속하지 않는다. 따라서 첫 출시는 SDK 기능을 억지로 추가하지 않고 STOVE 공식
가이드가 제공하는 간편 업로드/DRM 경로를 우선 사용한다. 공식 가이드는 SDK를
연동하지 않는 DRM Free 경로를 사용할 경우 STOVE 업적·인앱 결제·DLC 같은 기능을
쓸 수 없다고 안내한다.

사운드 추가가 끝나기 전에는 다음 작업을 하지 않는다.

- 출시 후보 Windows 빌드 생성
- STOVE Uploader 업로드
- 실행 파일·작업 폴더·실행 인자 확정
- 빌드 심사 요청

사운드 확정 뒤에는 태그 기반 Windows 패키지를 만들고 STOVE 클라이언트에서
설치·실행·종료·재실행·삭제를 확인한다. STOVE 전용 코드나 SDK가 실제로 필요해질
때만 별도 기능 브랜치에서 추가한다.

## 5. 자체등급분류 제출물

STOVE 자체등급은 전체이용가·12세이용가·15세이용가만 가능하다. 최종 내용이
청소년이용불가에 해당하면 STOVE 자체심의가 아니라 관할 등급분류기관 직접 심의
경로를 써야 한다.

현재 공식 체크리스트에 맞춰 다음 자료를 준비한다.

- 게임 설명서: 줄거리, 주요 인물, 조작법, 스크린샷 5장 이상, 치트 사용법(없으면
  없음으로 기재)
- 영상: 초반·중반·후반을 각각 7분 이상, 다중 엔딩 전체, 폭력·공포 등 등급 요소가
  잘 보이는 구간
- 게임 내 모든 CG/일러스트 파일 또는 대용량 다운로드 링크
- 캐릭터 대사, 내레이션, UI를 포함한 언어 파일
- 심의 대상으로 지정할 실제 출시 빌드
- 폭력성, 선정성, 공포, 언어, 약물, 범죄, 사행성 설문 답변

현재 자료 위치는 `stove/ratings/`이다. 영상과 최종 빌드만 사운드 확정 뒤 만든다.
STOVE 가이드는 사업자/은행 승인과 빌드 업로드 후 심의 등록을 진행하며, 자체심의와
빌드 검토에 약 7일을 안내한다. 혼잡하거나 보완 요청이 있으면 더 길어질 수 있다.

## 6. 가격·수수료·정산

- 현재 공개된 STOVE 유료 서비스 계약 샘플은 STOVE 수수료를 조정된 순매출의
  15%로 규정한다.
- 세금, 환불, 할인, 지급 조건과 통화는 실제 가입 화면의 최신 계약이 우선한다.
- 국내 유료 상품 정산은 공식 가이드의 지급 주기와 필요한 세금계산/증빙을 다시
  확인하고, 해외 플랫폼 매출과 함께 세무사에게 신고 방식을 확인한다.
- 가격은 Steam과 STOVE의 수수료·할인 정책을 비교한 뒤 정하되 이유 없는 플랫폼별
  가격 차이는 피한다.

## 7. STOVE 등급을 Steam에 사용

STOVE가 등급을 확정하고 게임이 실제 출시된 뒤 다음 순서로 진행한다.

1. 등급분류번호, 등급, 결정일, 게임명, 빌드 동일성을 확인한다.
2. 게임물관리위원회의 최신 `자체등급분류 게임물 유통 통보서` 양식을 받는다.
3. 유통 플랫폼을 Steam으로 적고 게임·사업자/개인·등급 정보를 작성한다.
4. 필요한 등급결정 자료와 Steam 상품 정보를 첨부해 제출한다.
5. 접수·보완 완료 기록을 받은 뒤 Steam 한국 판매 설정과 등급 표시를 최종 확인한다.

STOVE 공식 창작자 안내는 자체등급분류사업자가 아닌 플랫폼에 유통할 때
게임산업법 제21조의3 제2항 제2호에 따른 유통 통보서 제출로 등급을 사용할 수
있다고 설명한다. 반대로 다른 자체등급분류사업자 플랫폼에는 그대로 가져갈 수
없다. 제출 시점에는 게임물관리위원회의 최신 양식과 담당 부서 안내를 다시 확인한다.
