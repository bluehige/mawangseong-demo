# 사용자가 직접 해야 하는 STOVE 입점 작업

제가 대신 동의·서명·본인인증할 수 없는 항목만 순서대로 정리했다. 민감한 자료는
채팅이나 저장소에 보내지 않는다.

## 지금 할 일

1. <https://studio.onstove.com>에 장기간 사용할 대표 이메일 계정으로 로그인한다.
2. `Studio 시작하기`에서 최신 이용약관과 유료 서비스 조건을 읽고 직접 동의한다.
3. 그룹을 만들기 전에 아래 네 가지를 정한다.
   - 공개 개발자/퍼블리셔명
   - 그룹명
   - 영문 그룹 ID
   - 개인 또는 개인/법인 사업자 중 계약 주체
4. `stove/store/STUDIO_PORTAL_VALUES.md`의 프로젝트·상품 ID 추천안을 확인한다.
   ID는 생성 후 변경이 어렵기 때문에 승인 전 임의 생성하지 않는다.
5. 개인/사업자와 은행 정보 등록에 필요한 서류를 준비한다.
   - 개인: 3개월 이내 주민등록본 또는 여권, 통장 사본
   - 사업자: 사업자등록증, 통장 사본
6. 법적 이름, 예금주, 제출 서류의 표기가 일치하는지 확인하고 Studio에서 직접
   업로드한 뒤 심사를 요청한다.

## 개인 명의 선택 전 확인

STOVE는 개인 등록을 허용하지만 유료 판매자의 법정 표시정보가 상품 페이지에
노출될 수 있다. 현재 등록 화면에서 이름·주소·연락처 등 실제 공개 범위를 확인한
뒤 선택한다. 계속 상업 판매할 계획이거나 자택 주소 공개가 부담되면 다음을 먼저
확인한다.

- 홈택스/세무서 또는 세무사: 개인사업자 등록, 업종과 해외·국내 플랫폼 매출 신고
- 관할 시·군·구: 게임제작업/배급업과 통신판매업 신고 해당 여부
- 계좌: 계약 주체와 같은 예금주 계좌 사용 가능 여부

Steam을 개인 명의로 진행 중이어도 STOVE의 계약 주체는 별도로 결정할 수 있다.
다만 같은 게임의 권리 보유자와 정산 주체 관계를 설명할 수 있어야 한다.

## 저에게 알려줄 값

아래 값은 공개 또는 운영용 정보다. 비밀번호, 인증번호, 신분증, 주민등록번호,
계좌번호, 통장 사본은 알려주지 않는다.

- Studio 가입 완료 여부
- 그룹명과 그룹 ID
- 프로젝트 ID와 게임/상품 ID
- 공개 개발자명·퍼블리셔명
- 고객지원 이메일과 지원 페이지 주소
- 개인/사업자 중 선택한 유형(서류 내용은 보내지 않음)
- 사업자·은행 정보의 승인/반려 상태
- 목표 가격과 출시 시기

## 사운드 추가 뒤 직접 확인할 일

- 제가 만든 최종 Windows 빌드의 게임명·버전·언어를 확인
- STOVE Uploader 로그인과 업로드 승인
- STOVE 클라이언트에서 설치·실행·재실행·삭제
- 초반·중반·후반과 모든 엔딩 영상이 실제 빌드와 일치하는지 확인
- 자체등급 설문 답변과 예상 등급 최종 확인
- 상품 페이지, 가격, 공개일, 자동/수동 출시 설정 승인
- 검수 보완 요청과 최종 출시 버튼 확인

## STOVE 출시 뒤 직접 할 일

- 등급분류번호와 등급결정 내용을 내려받아 보관
- 게임물관리위원회 최신 양식으로 Steam 유통 통보 제출
- 접수 또는 보완 완료 기록 보관
- Steam 한국 판매 페이지의 등급 표시와 게임 빌드 동일성 최종 확인

문제가 생기면 화면 전체를 캡처해 보내되 주민등록번호, 계좌번호, 이메일 인증코드,
주소 등 민감정보는 먼저 가린다.
