# 작업 로그: 직접 조종 보정

작성일: 2026-07-02

## 요청

- 사용자가 직접 선택해서 개별 조종하려 했지만 원하는 대로 움직이지 않음.
- 실제로 선택한 몬스터를 직접 조종할 수 있게 확인하고 수정.

## 확인한 문제

- 직접 조종 시작 버튼은 `direct_control = true`만 설정하고, 기존 AI 이동 경로 `path_points`를 지우지 않았다.
- 사용자가 직접 조종을 켰을 때 이전 AI 목표가 남아 있으면 첫 명령 전까지 몬스터가 계속 AI 경로를 따라갈 수 있었다.
- 우클릭은 항상 지점 이동으로만 처리되어, 적을 찍어도 적 유닛 자체가 아니라 그 순간의 좌표만 따라갔다.
- 적이 이동하면 몬스터가 지정 좌표로만 가서 멈추거나, 충돌/사거리 상황에 따라 사용자가 기대한 "적을 따라가서 붙어 싸우기"처럼 보이지 않을 수 있었다.
- 전투 패널 위 우클릭도 맵 이동 명령으로 들어갈 수 있어 조작 오입력이 생길 수 있었다.

## 구현 내용

- `scripts/units/Unit.gd`
  - `command_target`을 추가해 직접 조종 공격 대상을 저장.
  - `begin_direct_control()`을 추가해 직접 조종 시작 시 기존 AI 경로, 이동 지점, 공격 대상, 우회 지점을 초기화.
  - `command_move(point)`는 이동 지점을 지정하고 기존 공격 대상을 해제.
  - `command_attack(unit)`은 적 유닛을 직접 공격 대상으로 지정하고 계속 추적.
  - 직접 조종 중 공격 대상이 죽거나 해제되면 자동으로 명령 대상을 지움.
  - 다운되거나 AI 복귀 시 직접 조종 명령 상태를 함께 초기화.

- `scripts/game/GameRoot.gd`
  - 전투 화면 우클릭 처리에서 UI 패널 위 입력은 무시.
  - 적 유닛 근처를 우클릭하면 지점 이동 대신 직접 공격 대상으로 지정.
  - 적이 아닌 맵 바닥을 우클릭하면 기존처럼 직접 이동 명령을 수행.

- `scripts/game/CombatSceneController.gd`
  - 직접 조종 시작 시 `begin_direct_control()`을 호출하도록 변경.
  - 직접 조종 중 수동 지정 대상이 사거리 안에 있으면 자동 타깃보다 먼저 공격.
  - 전투 로그 문구를 "우클릭 이동, 적 우클릭 공격 지정"으로 명확히 변경.

- `tools/DemoSmokeTest.gd`
  - 직접 조종 시작 시 기존 AI 이동 경로가 비워지는지 검증.
  - 적 우클릭 시 `command_target`이 지정되고 `command_point`가 비워지는지 검증.
  - 적을 직접 공격 대상으로 지정한 뒤 몬스터가 실제로 거리를 좁히는지 검증.

## 남은 판단 사항

- 실제 플레이에서 선택된 몬스터와 명령 대상이 충분히 눈에 띄는지 확인해야 한다.
- 우클릭 공격 지정 시 커서, 타깃 링, 짧은 명령 이펙트가 있으면 조작감이 더 명확해질 수 있다.
- 현재 UI 패널 차단 영역은 고정 좌표 기반이다. 전투 UI 배치가 크게 바뀌면 `_combat_ui_at()`도 같이 갱신해야 한다.
- 직접 조종 몬스터가 적에게 붙을 때 정지 거리는 `attack_range * 0.82`와 최소 12px 기준이다. 근접 전투가 너무 겹쳐 보이거나 멀어 보이면 여기부터 조정한다.
