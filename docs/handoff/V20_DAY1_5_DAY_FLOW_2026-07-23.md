## 2. 플레이어가 실행하는 DAY 흐름

하루에 노출하는 상태는 아래 다섯 개뿐이다.

1. `INTRUSION_BRIEF`: 플레이어가 침입 정보와 오늘의 전략 질문을 읽고 `배치 시작`을 누른다.
2. `PLACEMENT`: 같은 보드에서 시설을 설치·제거·이동·교체하고 `slime`, `goblin`, `imp`를 monster slot에 배치한다.
3. `DEFENSE_START`: 추가 확인창 없이 전투 직전 snapshot을 저장하고 3초를 센다. 플레이어가 취소하면 `PLACEMENT`로 돌아간다.
4. `COMBAT`: snapshot의 배치, seed와 RNG 상태로 전투를 시작한다. 전투 종료 callback 전에는 결과로 이동하지 않는다.
5. `RESULT`: 승리하면 DAY 1~4에서 다음 DAY의 `INTRUSION_BRIEF`로 이동한다. 패배하면 `같은 배치 재도전` 또는 `배치 수정` 중 하나를 선택한다.

허용 edge는 정확히 여덟 개다.

| from | to | 통과 조건 |
|---|---|---|
| `INTRUSION_BRIEF` | `PLACEMENT` | `배치 시작` action |
| `PLACEMENT` | `DEFENSE_START` | 배치 유효성 PASS |
| `DEFENSE_START` | `COMBAT` | snapshot 저장, 공간 해석, 3초 countdown 완료 |
| `DEFENSE_START` | `PLACEMENT` | 플레이어 취소 또는 시작 오류 |
| `COMBAT` | `RESULT` | 실제 전투 종료 callback |
| `RESULT` | `INTRUSION_BRIEF` | DAY 1~4 승리 |
| `RESULT` | `PLACEMENT` | 패배 뒤 `배치 수정` |
| `RESULT` | `DEFENSE_START` | 패배 뒤 `같은 배치 재도전` |

같은 상태 재진입과 나머지 17개 상태 쌍은 거부한다. DAY 5 승리는 다음 DAY를 만들지 않는다.

## 3. 배치·자원·재도전 규칙

`방어 시작`은 다음 네 조건을 모두 만족할 때만 활성화한다.

- 설치한 시설의 `cost` 합계가 10 이하다.
- `slime`, `goblin`, `imp`가 각각 비어 있지 않은 monster slot 하나를 가진다.
- 세 monster slot ID가 서로 다르다.
- 한 zone에 배치한 몬스터가 2명을 넘지 않는다.

`available_build_points`는 항상 `10 - sum(installed_facility.cost)`로 다시 계산한다. 시설 제거·이동·교체 뒤 저장된 잔액을 신뢰하지 않는다. 몬스터 배치는 건설 점수를 쓰지 않는다.

전투 직전 snapshot은 placement, 마왕 HP·mana, 명령 point·충전·cooldown, 몬스터 HP·mana·status·cooldown·level·EXP, 시설 charge·active·disable, `encounter_seed`, 전투 첫 physics frame 직전 RNG state를 포함한다.

- `같은 배치 재도전`: snapshot을 먼저 복원하고 편집 없이 `DEFENSE_START`에서 3초를 다시 센다.
- `배치 수정`: 같은 snapshot을 먼저 복원하고 `PLACEMENT`에서 시설과 몬스터 위치를 바꿀 수 있게 한다.
- 패배 salvage, 승리 income, 수리 reserve는 모두 0이다. 재도전 횟수와 승패로 건설 점수를 얻지 않는다.
- 새 DAY는 직전 배치를 유지하되 마왕 HP `1500/1500`, mana `320`, 명령 `3/3`, recharge `0`, 모든 cooldown `0`, 시설 charge max, active·disable `0`, 몬스터 level `1`, EXP `0`, HP·mana max, status `0`으로 시작한다.
- V20 전투에서 적 처치로 roster EXP를 지급하지 않는다.
