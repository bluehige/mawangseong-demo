# v0.2 출시 QA 보고서

- 대상 브랜치: `v.02`
- 검수일: 2026-07-13
- 기준 해상도: 1920x1080, 호환 해상도: 1366x768
- 자동 검증: 35개 출시 검증과 별도 UI 회귀 검증

## 출시 판정

핵심 캠페인, 저장/불러오기, 전투, 육성, 마왕성 확장, DAY 1~30 진행 및 엔딩 흐름은 출시 후보 수준으로 정리했다. 2026-07-13 최종 `Full` 자동 검증은 **35/35 PASS**이며, Web 내보내기와 브라우저 부팅/새 게임 입력도 통과했다.

## 관점별 검수

### 1. 일반 사용자

- 패배 후 진행이 멈추는 소프트락을 수정하고 결과/다음 진행 흐름을 복구했다.
- 유효한 저장이 있을 때 새 게임으로 덮어쓰지 않도록 확인 모달을 추가했다. 손상되거나 없는 저장은 새 게임을 바로 시작한다.
- 비활성 버튼에 빈 `Callable`이 연결되어 발생할 수 있는 입력 오류를 제거했다.
- 몬스터 화면의 `Esc` 복귀와 결과/튜토리얼 오버레이 정리를 즉시 반영하도록 수정했다.
- 저장 스키마에 교리 화면 상태, 지침 사용 이력, 누적 손실 지표를 포함하고 구버전 세이브 호환 검증을 추가했다.

### 2. 프로그래머

- 고속 시뮬레이션에서 경로 지점을 왕복하던 이동 판정을 프레임 이동량 기반 도달 반경으로 변경했다.
- 전투 속도 배율이 이동, 공격, 스킬 타이머에 일관되게 적용되도록 동기화했다.
- 방 단위 지침은 선택한 방의 유닛에만 적용하고, 대상 없는 화염 지대는 자원과 재사용 대기시간을 소비하지 않게 했다.
- 커스텀 레이아웃은 프로젝트 파일 대신 `user://`에 기록하도록 변경했다.
- 통합 검증 러너에 프로세스 종료 코드, 타임아웃, 산출물 최신성 검사를 추가했다.
- 런타임 온보딩 JSON을 `data/`로 이동해 문서 제외 규칙과 충돌하지 않게 했다.

### 3. 재미 중심 사용자

- 방어/총공격 지침이 생존과 처치 속도를 서로 다르게 바꾸는지 자동 비교한다.
- 감시소, 병영, 회복 둥지가 각각 감속/추가 피해, 전선 공방, 회복으로 다른 결과를 만들도록 검증한다.
- 전선 병영은 동일한 적 조합에서 중립 방의 패배를 승리로 뒤집었다.
- DAY 3 보스전은 보조 스킬을 사용하는 기준으로 약 57초이며, 출시 허용 구간을 32~65초로 설정했다.

### 4. 전투 재미 중심 사용자

- 아군 보호막형 구조 슬라임, 금고 특화 고블린, 지속 화염 지대 임프의 진화별 전투 동작과 AI를 구현했다.
- 도둑이 탈출한 경우 처치 수에 포함되지 않도록 수정했다.
- 병영의 공격/피해 감소 발동, 감시소의 추가 피해/감속, 영웅 돌진/최후의 맹세/용맹의 외침/왕실 집결을 계측한다.
- DAY 25~30 보스 및 공병 시나리오에서 시설 무력화, 지원 중단, 최종 보스 페이즈를 회귀 검증한다.

### 5. 디자인 중심 사용자

- 1920x1080에서 작게 보이던 유닛 스프라이트와 이름/체력 라벨을 확대했다.
- Stage 01 타일 배율을 `0.4911`, Stage 02~04를 `0.42`로 조정해 초기 성도 화면을 충분히 차지하게 했다.
- 22개 대형 건물 원본의 가져오기 최대 크기를 768로 제한해 실제 표시 크기와 GPU 메모리 사용을 맞췄다.
- 시설 선택 패널은 6개 항목이 잘리지 않도록 행 수를 수정했다.
- 1920x1080 단계별 성 캡처와 1366x768/텍스트 115%의 DAY 27~30 UI, 23줄 결산 스크롤, 엔딩 화면을 검사한다.

### 6. 몬스터 육성 사용자

- 훈련은 하루 횟수 제한과 최대 성장 한계를 적용해 무한 반복을 막았다.
- 진화 선택별 스킬과 AI가 실제 전투에서 다르게 작동하도록 연결했다.
- 전투 활동 경험치, 시설 기여, 집중 성장, 다음 날 준비 효과의 적용과 만료를 검증한다.
- 결과 화면 성장 카드의 상태/준비 효과/집중 버튼이 겹치지 않는지 모든 기본 몬스터에 대해 검사한다.

### 7. 스토리 중심 사용자

- DAY 29 결전 전야 10개 대사, 로만 보급 전언, 정식 레온 예고 및 전용 초상 렌더를 검증한다.
- DAY 29의 비전투 최종 준비 계약이 저장되고 DAY 30 시작 조건으로 이어지게 했다.
- 누적 보물/시설 손실과 사용 지침 이력을 엔딩 평가에 반영하고, 시설 평가는 실제 `facility_role` 기준으로 계산한다.
- 엔딩 후 계승 교리 화면 상태를 저장 주기에 포함했다.

- Godot 4.5.2 프로젝트 가져오기 및 스크립트 파싱

- 5개 플레이 성향 DAY 1~3 프록시와 15개 DAY 3 시작값

## 잔여 위험과 결정 항목

1. 공격형 자동 프록시는 DAY 3 시작값 3개에서 0/3 승리다. 총공격 지침의 고위험 성격을 유지한 결과이며 실제 사람 플레이 난이도 통계가 아니므로, 출시 후 플레이 로그에서 우선 관찰해야 한다.
2. 로컬 Godot에는 Web 내보내기 템플릿만 설치되어 있다. Windows 실행 파일을 이 환경에서 최종 생성하려면 약 1.26 GiB의 Godot 4.5.2 전체 템플릿 설치가 필요하다.
3. 자동 프록시와 이미지 경계 검사는 기능 회귀를 찾지만 사람의 손맛, 문장 취향, 장기 피로도를 완전히 대체하지 않는다.
