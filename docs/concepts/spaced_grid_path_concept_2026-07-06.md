# Spaced Grid Source Layout Concept - 2026-07-06

This document records the applied source contract for the novice Demon King castle layout. It supersedes the earlier packed `20x20` full-grid-object prototype.

## Applied Contract

- Contract ID: `novice_4x4_grid_5x5_gap2_paths_01`.
- Room lattice: `4x4`.
- One room slot: full `5x5` master cells.
- Gap between room slots: `2` master cells.
- Room stride: `[7, 7]`.
- Room lattice origin: `[2, 0]`.
- Room lattice size: `[26, 26]`.
- Active master size including west exterior approach: `[28, 26]`.
- Empty room slots are cave rock/void, not floor.
- Only explicitly placed corridor/outside modules produce walkable path floor.
- Every room edge is wall unless a connected paired socket opens it.
- Each room-to-path mouth is a paired two-cell opening.
- Entrance has an internal east opening and an exterior west opening.

## Required Connections

- `throne:S` to `spike_corridor:N` as two socket cells.
- `barracks:E` to `spike_corridor:W upper` as two socket cells.
- `recovery:W` to `spike_corridor:E upper` as two socket cells.
- `entrance:E` to `spike_corridor:W lower` as two socket cells.
- `entrance:W` to `outside_approach:E` as two socket cells.
- `treasure:W` to `spike_corridor:E lower` as two socket cells.
- `slot_01:N` to `spike_corridor:S` as two socket cells.

This produces `14` connection bridge records and `7` visible paired path mouths, including the exterior entrance mouth.

- `data/dungeon_quarter/starting_layout.json`: default layout uses the new spaced contract and includes `outside_approach`.
- `data/dungeon_quarter/custom_layouts.json`: sample custom layout is aligned to the same contract so layout switching does not fall back to the old packed grid.
- `data/dungeon_quarter/room_blueprints.json`: adds `corridor_gap_network_01` and `outside_approach_01`.
- `data/dungeon_quarter/castle_grade_rules.json`: max grid is `[28, 26]`.
- `data/dungeon_quarter/asset_manifest.json`: declares `spaced_grid_gap_path_connections_01` and the source concept image paths.
- `scripts/dungeon_quarter/ModuleGraph.gd`: grid size and facing-center calculation support the west exterior module.
- `scripts/dungeon_quarter/QuarterDungeonRenderer.gd`: draws the outside approach cells and a visible exterior cave-mouth overlay.
- `tools/QuarterModuleSmokeTest.gd`: verifies 28x26 grid size, 14 bridge records, 7 mouth groups, 4 outside approach cells, entrance `E,W` variant, and outside-to-throne pathing.

## Remaining Art Limitation

The structure is now proven in source. The room art is still a temporary procedural/full-grid fallback plus existing facing props. Production room assets still need the full matrix:

- room role,
- `NW/NE/SE/SW` object-front direction,
- 16-state `N/E/S/W` wall/open-mask,
- optional back/front layer split.

Do not mass-generate that matrix until the current spaced source layout and exterior entrance treatment are accepted.
