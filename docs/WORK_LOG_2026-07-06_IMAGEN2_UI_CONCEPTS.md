## 관리 화면 시안 평가

- 레퍼런스의 핵심 톤인 동굴 배경, 거친 돌벽, 보라색 마력광, 금속/금장 프레임이 잘 맞는다.
- 중앙 맵은 방과 길이 분리되어 보이며, 방이 작은 타일 묶음이 아니라 하나의 완성된 공간처럼 보인다.
- 입구, 왕좌의 방, 병영, 회복 둥지, 보물 보관실, 건설 가능 슬롯이 모두 한 화면에서 읽힌다.
- 상단 자원바, 좌측 메뉴, 우측 선택 방 정보, 하단 주요 액션 바의 정보 계층이 레퍼런스와 맞다.
- 실제 구현 시 단색 `Panel`을 쓰지 말고 이 시안의 프레임/버튼/자원 패널을 이미지 에셋으로 분리해야 한다.

## 전투 화면 시안 평가

- 방 안에서 전투가 일어나는 구조가 명확하다.
- 선택 유닛 패널, 전투 로그, 속도 버튼, 전체 지침 패널이 한 화면에 들어온다.
- 좌측 전체 지침 패널이 화면을 크게 차지하므로 실제 구현에서는 폭을 줄이거나 접힘/아이콘 모드가 필요하다.
- 하단 로그는 분위기가 좋지만 전투 시야를 가릴 수 있으므로 로그 높이와 투명도 조절이 필요하다.
- 우측 선택 유닛 패널은 구현 기준으로 쓸 수 있다. 초상, HP, 스탯, 스킬 슬롯 순서가 좋다.

## 구현 방향

- 다음 구현은 코드로 도형을 그리는 방식이 아니라, 생성 이미지 기준으로 UI 에셋을 분리해서 적용한다.
- 우선 분리 대상:
  - 상단 자원 패널
  - 좌측 세로 메뉴 버튼
  - 하단 대형 액션 버튼
  - 우측 인스펙터 프레임
  - 전투 로그 프레임
  - 지침/스킬 버튼 프레임
- 화면 배경은 관리/전투/결산/대화/이름 입력 화면별로 별도 이미지가 필요하다.
- 텍스트는 생성 이미지 안에 박지 말고 Godot Label로 올린다. 생성 이미지는 프레임, 배경, 아이콘, 장식만 담당한다.
- 방/길 구조는 현재 그리드 시스템을 유지하되, 방 오브젝트와 벽/문/길 이미지는 생성 이미지 톤에 맞춰 재생성한다.

## 전투 UI 미확정 사항

- 전체 지침은 좌측 대형 패널형과 하단 소형 버튼형 중 하나를 선택해야 한다.
- 직접 조종 모드일 때만 스킬 슬롯을 크게 보여줄지, 항상 우측 패널에 노출할지 결정이 필요하다.
- 로그는 하단 고정형, 접힘형, 최근 3줄 오버레이형 중 하나를 테스트해야 한다.
- 방 기반 전투를 강조하려면 유닛 HP바보다 방 상태/입구 돌파 여부/왕좌까지 남은 경로를 더 잘 보여줘야 한다.
