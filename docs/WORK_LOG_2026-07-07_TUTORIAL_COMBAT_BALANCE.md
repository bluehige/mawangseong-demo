# Work Log: Tutorial Combat Balance Pass (2026-07-07)

## 목표

튜토리얼 DAY1/DAY2 전투가 120초 제한에 걸리던 문제를 해결하고, 이후 스토리 진행과 침공 시스템까지 확장할 수 있는 밸런스 구조를 마련했다.

초보자 설명:

- 웨이브는 한 번에 들어오는 적 묶음이다.
- 스케일은 기본 능력치에 곱하는 배율이다.
- 교착은 전투가 사실상 멈춰서 승패가 나지 않는 상태다.

## 이번 결정

튜토리얼 전투는 `지휘형 오토배틀 + 몬스터 육성형 + 안정형 밸런스` 방향으로 둔다.

즉, 초반에는 플레이어가 직접 조작을 완벽히 하지 않아도 전투가 끝나야 하고, 대신 지시/방 배치/스킬을 쓰면 더 좋은 결과가 나오는 방향으로 설계한다.

## 구현 내용

- 유닛 길찾기 도착 판정 반경을 `6px/8px`에서 `12px`로 정리했다.
- 방 중심 기준 경로 대신 현재 월드 위치 기준 경로를 쓰도록 바꿨다.
- 같은 방의 적을 추격할 때는 복잡한 방 경로 대신 직접 목표 지점으로 이동하게 했다.
- 방어 지시는 배치 방과 인접 방 중심으로 제한해, 몬스터 수가 늘어났을 때 전체 던전이 한 덩어리로 몰려다니지 않게 했다.
- 함정 유도 중에도 같은 방에 적이 있으면 직접 교전하게 했다.
- `data/waves.json`에 `hp_scale`, `atk_scale`, `spawn_interval`을 추가로 쓸 수 있게 했다.
- `tools/BalanceSimulation.gd -- --assert-tutorial-balance` 자동 검증을 추가했다.

## 향후 확장 밸런스 선택지

### 선택지 A: 챕터별 침공 티어

스토리 챕터마다 침공 티어를 올린다.

- 장점: 난이도 상승을 플레이어가 이해하기 쉽다.
- 단점: 챕터 밖 반복 플레이의 다양성은 부족할 수 있다.
- 적용 방법: `chapter_1`, `chapter_2`, `invasion_tier_1` 같은 별도 보정값을 웨이브 데이터에 붙인다.

추천도: 높음. 스토리가 이어지는 구조와 잘 맞는다.

### 선택지 B: 적 역할별 강화

적 전체 체력만 올리지 않고, 역할별로 강화 방향을 다르게 둔다.

- 탐험가: 기본 전열, 체력과 방어 상승.
- 도둑: 속도와 목표 압박 상승.
- 용사/보스: 스킬 빈도와 순간 위협 상승.

장점: 단순 피통 싸움이 줄고, 대응 전략이 생긴다.
단점: AI와 UI 설명이 조금 더 필요하다.

추천도: 높음. 재미 요소를 늘리기에 가장 좋다.

### 선택지 C: 몬스터 성장 대응형 난이도

플레이어 몬스터 레벨이나 보유 몬스터 수에 따라 침공 강도를 조정한다.

- 장점: 플레이어가 강해져도 게임이 너무 쉬워지지 않는다.
- 단점: 과하면 플레이어가 성장 보상을 못 느낀다.

추천도: 중간. 메인 스토리 이후 반복 침공에 적합하다.
