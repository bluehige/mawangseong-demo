# 정규 캠페인 저장·이어하기 인수인계 (2026-07-12)

## 최종일 재확인

정규 1.0 캠페인의 최종일은 **DAY 30**이다. DAY 40을 최종일로 정한 근거는 현재 저장소와 도달 가능한 Git 전체 이력에서 발견되지 않았다.

- `scripts/game/GameRoot.gd`: `REGULAR_CAMPAIGN_FINAL_DAY = 30`
- `data/campaign_days.json`: 마지막 캠페인 항목은 `day_30`, `final_battle = true`
- `data/waves.json`: 마지막 웨이브는 `day_30`
- `docs/regular_version/REGULAR_VERSION_TARGET_SUMMARY.md`: 정규 캠페인 30일
- `docs/STORY_PLAN_AFTER_TUTORIAL_2026-07-08.md`: 최종장 DAY 26~30, 최종전·엔딩 DAY 30

`GameState.max_day = 3`은 정규 캠페인 상한이 아니라 DAY 1~3 온보딩 상한이다. 최근 문서의 “DAY 28~30 최종장”은 마지막 제작 묶음 이름이며, 전체 서사 구간은 DAY 26~30이다.

## 이번 제작 범위

제목 화면의 가짜 이어하기 위치에 있던 `빠른 시작`과 별도로 실제 `이어하기`를 추가했다. 빠른 시작은 기존 자동 검사와 개발 편의를 위해 별도 버튼으로 유지한다.

- 버전 1 저장 파일: `user://campaign_save_v1.json`
- 임시 파일과 이전 파일 백업을 거쳐 교체하고, 저장 도중 종료된 경우 `.tmp` 또는 `.bak`에서 자동 복구하는 방식
- 제목 화면 DAY·성 단계·마왕명·재개 화면 요약
- 손상 파일과 지원하지 않는 버전 감지, 복원 실패 표식, 이어하기 비활성화
- 자동 저장 실패를 발생한 화면 위에 즉시 표시하고, 이전 정상 저장은 유지
- 새 게임·빠른 시작·엔딩 새 게임에서 이전 저장 명시 삭제
- 앱을 종료해도 DAY 28 선택 대기, DAY 30 재도전, 엔딩과 후일담 복원
- Stage 04를 새 게임으로 되돌릴 때 왕좌 최대 체력이 1500으로 초기화되지 않던 문제 수정

신규 캐릭터나 그래픽은 추가하지 않았으므로 이번 작업에는 새 이미지 생성이 필요하지 않았다.

## 저장 범위

저장 파일에는 전투 중 좌표나 맵 편집 초안이 아니라, 다시 시작해도 모순이 생기지 않는 진행 상태만 기록한다.

- 날짜, 자원과 수입, 왕좌 현재·최대 체력, 마왕명
- 4단계 성 진화 이력, 11구역과 시설 강화, 선택한 사용자 맵 원본
- 몬스터 레벨·EXP·배치·승급·특화·다음 방어 준비
- 전체 지침과 방 지침
- 원정 선택·완료·보상·다음 방어 보정
- 챕터·최종장·패배 재도전·엔딩·후일담 플래그
- 전투 결산, 성장 검토와 집중 성장 선택
- 온보딩 단계, 완료한 안내, 본 대사, 진행 중 대사 위치

JSON에서 숫자 사전 키가 문자열로 바뀌는 문제를 막기 위해 본 일차 목록은 정수 배열로 저장하고 복원할 때 다시 정수 키로 만든다. 사용자 맵은 Web에서 `res://` 쓰기가 불가능해도 이어지도록 현재 레이아웃 원본을 저장 파일 안에 함께 넣는다.

## 자동 저장 지점

화면 전환 동작이 모두 끝난 다음 프레임에 아래 안전 화면을 저장한다.

- 성 관리와 몬스터 관리
- 원정 예고와 원정 확정
- 캠페인 대화
- 전투 결산과 성장 선택
- 성 진화가 반영된 결산
- 최종 엔딩과 후일담 관리

전투 중과 맵 편집 초안 상태에서는 저장하지 않는다. 전투 도중 강제 종료하면 마지막 전투 직전 안전 지점으로 돌아가므로, 이미 소모한 원정 보정이나 보상을 중복 적용하지 않는다.

## 복원 순서

1. 버전·최종일·안전 화면·필수 자료와 내부 자료형을 검증한다.
2. DAY·성 단계·해금 플래그·최종 준비·엔딩·후일담 조합과 입구→왕좌 경로를 검증한다.
3. 복원 중 자동 저장을 잠근다.
4. 기존 전투 유닛과 효과, 맵 편집 초안을 정리한다.
5. 날짜·자원·왕좌와 성 단계·방·사용자 맵을 복원한다.
6. 로스터·지침·원정·캠페인·결산 상태를 복원한다.
7. 온보딩과 진행 중 대사를 복원한다.
8. 던전 그래프와 렌더러를 갱신한 뒤 저장된 안전 화면을 연다.

## 다음 제작 우선순위

사용자 지시에 따라 다음 구현은 시작하지 않고 **대기**한다. 다음 제작 방향은 **다회차 플레이, 여러 엔딩, 몬스터 장기 육성**이다. 기존 Web 최적화 우선순위보다 이 기획을 먼저 정리한다.

다음 작업을 재개할 때 먼저 확정할 항목은 다음과 같다.

1. DAY 30 첫 클리어 뒤 `새 회차`로 넘어가는 진입점과 회차 번호
2. 엔딩 분기 조건표: 성 운영, 원정 선택, 보안 평가, 동료·몬스터 성장 결과가 어떤 엔딩에 영향을 주는지

4. 반복 육성의 상한과 난이도 상승 규칙: 무한 수치 증가 대신 새 빌드와 선택지가 열리도록 설계
