# 작업 로그: 방 용도 변경 시스템

작성일: 2026-07-02

## 요청

사용자가 "이제 방용도 변경 시스템 구축해봐"라고 요청했다.

이전까지는 방 용도 변경이 설계 우선순위로 남아 있었고, 건설 슬롯은 하단 `건설` 버튼으로 감시 초소만 만들 수 있었다. 이번 작업의 목표는 관리 화면에서 선택한 방의 용도를 직접 바꾸고, 전투 로직도 바뀐 용도를 실제 기준으로 사용하게 만드는 것이다.

## 구현 내용

- `GameRoot.gd`
  - 각 방에 `facility_role` 런타임 필드를 초기화
  - 변경 가능 시설 목록 추가:
    - 병영
    - 보물고
    - 회복
    - 감시
    - 빈 슬롯
  - 입구, 가시 복도, 중앙 통로, 왕좌의 방은 고정 시설로 변경 불가 처리
  - 시설 변경 비용 처리 추가
  - 보물고와 회복은 고유 시설로 처리해서 새 위치로 옮기면 기존 위치는 빈 슬롯으로 전환
  - 빈 슬롯 또는 배치 한도 초과가 된 방의 몬스터를 자동 재배치
  - 현재 시설 역할과 방 타입을 조회하는 헬퍼 추가

- `HUDController.gd`
  - 선택 방 패널에 `시설 변경` 영역 추가
  - 병영, 보물고, 회복, 감시, 빈 슬롯 버튼 추가
  - 현재 용도 버튼은 비활성화 및 강조 표시
  - 비용 안내 문구 추가
  - 방 지침 버튼을 하단으로 재배치

- `CombatSceneController.gd`
  - 도둑의 목표 방을 현재 보물고 위치로 조회
  - 보물고가 없으면 도둑은 왕좌 목표로 동작
  - 약탈 판정도 현재 보물고 위치 기준으로 변경
  - 회복 후퇴와 회복 효과를 현재 회복 시설 위치 기준으로 변경
  - 왕좌 공격 판정은 현재 core 타입 방 기준으로 조회

- `TargetingService.gd`
  - 몬스터 우선 타깃 판단에서 core 방과 보물고 방을 파라미터로 받도록 변경
  - 도둑이 현재 보물고 또는 그 인접 방에 있을 때 우선순위를 높이도록 동적 계산

- `DemoSmokeTest.gd`
  - 병영을 보물 보관실로 변경하는 테스트 추가
  - 기존 보물 보관실이 빈 슬롯으로 전환되는지 검증
  - 비용이 차감되는지 검증
  - 도둑 목표가 새 보물 보관실을 추적하는지 검증

- `ManualVerificationCapture.gd`
  - 첫 관리 화면 캡처를 병영 선택 상태로 찍어 시설 변경 버튼을 확인할 수 있게 변경

## 검증

실행한 명령:

```powershell
godot --headless --path . --import
godot --headless --path . --scene res://tools/DemoSmokeTest.tscn
godot --path . --scene res://tools/ManualVerificationCapture.tscn
```

결과:

- Godot import 성공
- `DemoSmokeTest.tscn` PASS
- 수동 검증 캡처 생성 성공
- `tmp/manual_verification/01_management.png`에서 시설 변경 버튼, 비용 문구, 방 지침 버튼이 겹치지 않는 것을 확인

## 남은 검수 포인트

- 실제 플레이에서 방 용도 변경 비용이 너무 싸거나 비싼지 확인 필요
- 보물고와 회복을 옮길 때 기존 위치가 빈 슬롯으로 바뀌는 규칙이 사용자에게 충분히 명확한지 확인 필요
- 용도 변경 때문에 몬스터가 자동 재배치됐을 때 별도 안내 UI가 필요한지 판단 필요
- 저장/불러오기 시스템이 아직 없으므로 용도 변경은 현재 세션 런타임 상태에만 남는다.
